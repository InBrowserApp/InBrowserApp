## מהו קוד מצב HTTP?

קוד מצב HTTP הוא קוד תגובה בן שלוש ספרות שהשרת מחזיר כדי להראות מה קרה לבקשה. רואים אותו לעיתים קרובות בכלי המפתחים בדפדפן, בתגובות API, בלוגים של השרת, בבדיקות זמינות ובלוחות בקרה של reverse proxy.

### איך קוראים את משפחות הקודים העיקריות

- **1xx אינפורמטיבי:** השרת קיבל את הבקשה והעיבוד עדיין נמשך.
- **2xx הצלחה:** הבקשה הושלמה בהצלחה.
- **3xx הפניה מחדש:** הלקוח צריך לעקוב אחרי מיקום אחר או להשתמש שוב בתוצאה ממטמון.
- **4xx שגיאת לקוח:** יש בעיה בבקשה עצמה, למשל משאב חסר, קלט לא תקין או אימות שנכשל.
- **5xx שגיאת שרת:** השרת או תלות במעלה הזרם נכשלו בזמן עיבוד בקשה תקינה.

### מתי ה lookup הזה שימושי

השתמשו בכלי הזה כשצריך לאשר מה משמעותו של קוד מסוים, להשוות בין קודים דומים כמו 401 מול 403 או 502 מול 504, או לחפש לפי ביטוי שהופיע בהודעת שגיאה. החיפוש תומך בקוד, בשם הסטטוס ובתיאור מקומי.

### למה פרשנות נכונה חשובה

בזמן ניפוי תקלות, קוד המצב הוא לעיתים קרובות הרמז המהיר ביותר. תגובת 4xx מצביעה בדרך כלל על הבקשה, פרטי ההזדהות או משאב היעד. תגובת 5xx מצביעה בדרך כלל על האפליקציה, השער או שירות upstream. קריאת הקטגוריה תחילה עוזרת לבחור את הצעד הבא הנכון.
