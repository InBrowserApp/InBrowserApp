## HTTP Status Code คืออะไร?

HTTP status code คือรหัสตอบกลับแบบสามหลักที่เซิร์ฟเวอร์ส่งกลับมาเพื่อบอกว่าเกิดอะไรขึ้นกับคำขอหนึ่งรายการ คุณจะเห็นรหัสเหล่านี้บ่อยใน developer tools ของเบราว์เซอร์, การตอบกลับจาก API, server logs, ระบบตรวจสอบ uptime และแดชบอร์ด reverse proxy.

### วิธีอ่านกลุ่มรหัสสถานะหลัก

- **1xx Informational:** เซิร์ฟเวอร์ได้รับคำขอแล้ว และกระบวนการยังดำเนินต่ออยู่
- **2xx Success:** คำขอเสร็จสมบูรณ์เรียบร้อย
- **3xx Redirection:** ไคลเอนต์ต้องไปยังตำแหน่งอื่นหรือใช้ผลลัพธ์จากแคชซ้ำ
- **4xx Client Error:** ปัญหาอยู่ที่ตัวคำขอเอง เช่น ไม่พบทรัพยากร, ข้อมูลไม่ถูกต้อง หรือการยืนยันตัวตนล้มเหลว
- **5xx Server Error:** เซิร์ฟเวอร์หรือบริการ upstream ล้มเหลวระหว่างจัดการคำขอที่ถูกต้อง

### เมื่อไรที่ lookup นี้มีประโยชน์

ใช้เครื่องมือนี้เมื่อคุณต้องการยืนยันความหมายของรหัสหนึ่งตัว, เปรียบเทียบรหัสที่ใกล้กันอย่าง 401 กับ 403 หรือ 502 กับ 504, หรือค้นหาจากวลีที่เห็นในข้อความผิดพลาด เครื่องมือนี้ค้นหาได้ทั้งตามรหัส, ชื่อสถานะ และคำอธิบายที่แปลตามภาษา

### ทำไมการตีความให้ถูกต้องจึงสำคัญ

ระหว่างการดีบัก status code มักเป็นเบาะแสที่เร็วที่สุด คำตอบ 4xx มักชี้ไปที่คำขอ, ข้อมูลรับรอง หรือทรัพยากรเป้าหมาย ส่วน 5xx มักชี้ไปที่แอปพลิเคชัน, เกตเวย์ หรือบริการ upstream การดูหมวดหมู่ก่อนช่วยให้เลือกขั้นตอนถัดไปได้แม่นขึ้น
