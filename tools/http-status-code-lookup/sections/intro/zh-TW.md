## 什麼是 HTTP 狀態碼？

HTTP 狀態碼是伺服器回傳的三位數回應碼，用來說明一次請求最後發生了什麼。你會在瀏覽器開發者工具、API 回應、伺服器日誌、可用性監控和反向代理面板中經常看到它們。

### 如何理解主要的狀態碼類別

- **1xx 資訊性：** 伺服器已收到請求，處理流程仍在進行中。
- **2xx 成功：** 請求已成功完成。
- **3xx 重新導向：** 用戶端需要前往其他位置，或重用快取結果。
- **4xx 用戶端錯誤：** 請求本身有問題，例如資源不存在、輸入無效或驗證失敗。
- **5xx 伺服器錯誤：** 伺服器或上游依賴在處理有效請求時發生故障。

### 什麼時候適合使用這個 Lookup

當你需要確認某個狀態碼的意思、比較相近狀態碼之間的差異，例如 401 和 403、502 和 504，或想用錯誤訊息中的關鍵字搜尋時，這個工具就很適合。它支援依狀態碼、狀態名稱和在地化描述進行查找。

### 為什麼正確解讀很重要

在除錯時，狀態碼通常是最快的線索。4xx 通常表示問題在請求、憑證或目標資源；5xx 通常表示問題在應用程式、閘道或上游服務。先看懂類別，再決定下一步，會有效率得多。
