## 什么是 HTTP 状态码？

HTTP 状态码是服务器返回的三位响应代码，用来说明一次请求最终发生了什么。你会在浏览器开发者工具、API 响应、服务器日志、可用性监控和反向代理面板里经常看到它们。

### 如何理解主要的状态码类别

- **1xx 信息性：** 服务器已经收到请求，处理流程仍在继续。
- **2xx 成功：** 请求已经成功完成。
- **3xx 重定向：** 客户端需要跳转到其他位置，或者复用缓存结果。
- **4xx 客户端错误：** 请求本身存在问题，例如资源不存在、输入无效或鉴权失败。
- **5xx 服务器错误：** 服务器或上游依赖在处理一个有效请求时发生了故障。

### 什么时候适合使用这个 Lookup

当你需要确认某个状态码的含义、比较相近状态码之间的差异，例如 401 和 403、502 和 504，或者想按错误描述中的关键词来搜索时，这个工具就很合适。它支持按状态码、状态名称和本地化描述进行检索。

### 为什么正确解读很重要

在排障时，状态码往往是最快的线索。4xx 通常意味着问题出在请求、凭证或目标资源上；5xx 则通常意味着应用、网关或上游服务出了问题。先读懂类别，再决定下一步，会更高效。
