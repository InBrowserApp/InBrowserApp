## HTTP स्टेटस कोड क्या है?

HTTP स्टेटस कोड तीन अंकों वाले रिस्पॉन्स कोड होते हैं जिन्हें सर्वर यह बताने के लिए लौटाता है कि किसी अनुरोध के साथ क्या हुआ। आप इन्हें ब्राउज़र डेवलपर टूल्स, API रिस्पॉन्स, सर्वर लॉग, अपटाइम मॉनिटर और रिवर्स प्रॉक्सी डैशबोर्ड में अक्सर देखेंगे।

### मुख्य स्टेटस कोड परिवारों को कैसे पढ़ें

- **1xx जानकारीपरक:** सर्वर ने अनुरोध प्राप्त कर लिया है और प्रक्रिया अभी जारी है।
- **2xx सफलता:** अनुरोध सफलतापूर्वक पूरा हो गया।
- **3xx रीडायरेक्शन:** क्लाइंट को किसी दूसरे स्थान का अनुसरण करना है या कैश किया गया परिणाम फिर से उपयोग करना है।
- **4xx क्लाइंट त्रुटि:** समस्या स्वयं अनुरोध में है, जैसे संसाधन न मिलना, अमान्य इनपुट या प्रमाणीकरण विफल होना।
- **5xx सर्वर त्रुटि:** सर्वर या कोई अपस्ट्रीम निर्भरता एक वैध अनुरोध संसाधित करते समय विफल हो गई।

### यह Lookup कब उपयोगी है

इस टूल का उपयोग तब करें जब आपको किसी कोड का अर्थ पक्का करना हो, 401 और 403 या 502 और 504 जैसे मिलते-जुलते कोड की तुलना करनी हो, या किसी एरर संदेश के वाक्यांश से खोज करनी हो। यह कोड, स्टेटस नाम और स्थानीयकृत विवरण के आधार पर खोज करता है।

### सही व्याख्या क्यों महत्वपूर्ण है

डिबगिंग के दौरान स्टेटस कोड अक्सर सबसे तेज सुराग होता है। 4xx रिस्पॉन्स आमतौर पर अनुरोध, क्रेडेंशियल्स या लक्ष्य संसाधन की ओर इशारा करता है, जबकि 5xx रिस्पॉन्स आमतौर पर एप्लिकेशन, गेटवे या अपस्ट्रीम सेवा की ओर इशारा करता है। पहले श्रेणी समझने से अगला सही कदम जल्दी चुना जा सकता है।
