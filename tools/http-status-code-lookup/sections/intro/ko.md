## HTTP 상태 코드란 무엇인가요?

HTTP 상태 코드는 요청에 대해 어떤 일이 일어났는지 보여 주기 위해 서버가 반환하는 세 자리 응답 코드입니다. 브라우저 개발자 도구, API 응답, 서버 로그, 모니터링 도구, 리버스 프록시 대시보드에서 자주 보게 됩니다.

### 주요 상태 코드 범주 읽는 법

- **1xx 정보 응답:** 서버가 요청을 받았고 처리가 아직 진행 중입니다.
- **2xx 성공:** 요청이 정상적으로 완료되었습니다.
- **3xx 리디렉션:** 다른 위치를 따라가거나 캐시된 결과를 다시 사용해야 합니다.
- **4xx 클라이언트 오류:** 리소스 누락, 잘못된 입력, 인증 실패처럼 요청 자체에 문제가 있습니다.
- **5xx 서버 오류:** 유효한 요청을 처리하는 동안 서버 또는 업스트림 의존성이 실패했습니다.

### 이 Lookup 이 유용한 때

특정 상태 코드의 의미를 확인해야 할 때, 401 과 403, 502 와 504 처럼 비슷한 코드를 비교할 때, 또는 오류 메시지에 나온 문구로 검색하고 싶을 때 유용합니다. 이 도구는 코드, 상태 이름, 현지화된 설명으로 검색할 수 있습니다.

### 왜 정확한 해석이 중요한가요?

디버깅에서 상태 코드는 가장 빠른 단서인 경우가 많습니다. 4xx 응답은 보통 요청, 자격 증명, 대상 리소스 쪽 문제를 가리키고, 5xx 응답은 보통 애플리케이션, 게이트웨이, 업스트림 서비스 쪽 문제를 가리킵니다. 먼저 범주를 읽으면 다음 조치를 더 빨리 정할 수 있습니다.
