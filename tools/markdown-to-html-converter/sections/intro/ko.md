## 이 변환기를 사용하는 이유

Markdown은 문서, 릴리스 노트, 가벼운 콘텐츠 초안에 적합하지만 게시 대상은 보통 HTML을 요구합니다. 이 도구는 두 면을 동시에 보여 주므로 Markdown을 작성하거나 가져오고, 생성된 조각을 확인한 뒤 내보내기 전에 스타일이 적용된 미리보기를 점검할 수 있습니다.

## 여기서 할 수 있는 일

- Markdown을 직접 붙여넣거나 일반 텍스트 파일을 가져옵니다.
- 미리보기와 공유용 결과를 더 안전하게 만들고 싶을 때 정화를 켭니다.
- 렌더링된 미리보기와 정확한 HTML 조각을 동시에 검토합니다.
- 브라우저를 떠나지 않고 복사, 다운로드, 인쇄, 추가 수정까지 이어갑니다.

## 정화된 HTML과 원본 HTML

정화는 생성된 결과에서 안전하지 않은 태그와 속성을 제거합니다. 일반 게시 작업, CMS 블록, 이메일 스니펫, 공유 문서에는 켜 둔 상태를 권장합니다. Markdown 파서가 반환한 원본 조각을 꼭 확인해야 할 때만 끄세요.

## 대표적인 활용 사례

- README 일부를 임베드 가능한 HTML 블록으로 바꾸기.
- Markdown 초안에서 CMS용 콘텐츠 준비하기.
- 사이트 빌더에 붙여넣기 전에 생성된 마크업 검토하기.
- 검토나 승인용으로 스타일 적용된 미리보기 인쇄하기.
