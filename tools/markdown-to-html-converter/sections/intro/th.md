## ทำไมจึงควรใช้ตัวแปลงนี้

Markdown เหมาะมากสำหรับการร่างเอกสาร บันทึกการออกรุ่น และเนื้อหาที่กระชับ แต่ปลายทางที่ใช้เผยแพร่มักต้องการ HTML เครื่องมือนี้ทำให้คุณเห็นทั้งสองด้านพร้อมกัน คุณจึงพิมพ์หรือนำเข้า Markdown ตรวจสอบชิ้นส่วนที่สร้างขึ้น และดูตัวอย่างที่จัดรูปแบบก่อนส่งออกได้

## คุณทำอะไรได้บ้างที่นี่

- วาง Markdown โดยตรงหรือนำเข้าไฟล์ข้อความธรรมดา
- เปิดการกรองเมื่อคุณต้องการผลลัพธ์ที่ปลอดภัยมากขึ้นสำหรับการแสดงตัวอย่างและการแชร์
- ตรวจสอบตัวอย่างที่เรนเดอร์แล้วและชิ้นส่วน HTML ที่แม่นยำไปพร้อมกัน
- คัดลอก ดาวน์โหลด พิมพ์ หรือแก้ไขต่อได้โดยไม่ต้องออกจากเบราว์เซอร์

## HTML ที่ผ่านการกรองและ HTML ดิบ

การกรองจะลบแท็กและแอตทริบิวต์ที่ไม่ปลอดภัยออกจากผลลัพธ์ที่สร้างขึ้น ควรเปิดทิ้งไว้สำหรับการเผยแพร่ทั่วไป บล็อก CMS ชิ้นส่วนอีเมล และเอกสารที่แชร์ ปิดเฉพาะเมื่อคุณต้องการตรวจสอบชิ้นส่วนดิบที่ตัวแยก Markdown ส่งกลับมาโดยตรง

## กรณีใช้งานที่พบบ่อย

- แปลงส่วนต่าง ๆ ของ README ให้เป็นบล็อก HTML ที่ฝังได้
- เตรียมเนื้อหา CMS จากร่าง Markdown
- ตรวจสอบมาร์กอัปที่สร้างขึ้นก่อนวางลงในตัวสร้างเว็บไซต์
- พิมพ์ตัวอย่างที่จัดรูปแบบแล้วเพื่อการทบทวนหรืออนุมัติเนื้อหา
