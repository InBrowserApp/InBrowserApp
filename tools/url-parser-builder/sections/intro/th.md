## เครื่องมือนี้ทำอะไรได้บ้าง

ตัวแยกและสร้าง URL นี้ทำงานได้ทั้งสองทิศทาง คุณสามารถวาง URL แบบเต็มเพื่อดูโปรโตคอล ข้อมูลรับรอง โฮสต์ เส้นทาง พารามิเตอร์คิวรี และส่วนท้าย หรือจะแก้ทีละส่วนแล้วให้เครื่องมือประกอบที่อยู่สุดท้ายให้ใหม่ก็ได้

## ใช้เมื่อไหร่จึงเหมาะ

เหมาะเมื่อคุณต้องดีบักลิงก์ที่เสีย เตรียม API endpoint ตรวจสอบพารามิเตอร์แคมเปญ หรือจัดระเบียบ URL ยาว ๆ ก่อนแชร์ ตัวแก้ไขคิวรีแบบมีโครงสร้างมีประโยชน์มากเมื่อมีคีย์ซ้ำหรือช่องว่างที่ถูกเข้ารหัสจนการแก้ด้วยมือผิดพลาดได้ง่าย

## หมายเหตุในการใช้งาน

เครื่องมือนี้ทำงานได้ดีที่สุดกับ URL แบบสมบูรณ์และเป็น absolute เช่น `https://example.com/path` เบราว์เซอร์จะเข้ารหัสอักขระสงวนในผลลัพธ์ที่ปรับรูปแบบแล้วโดยอัตโนมัติ ทำให้ช่องว่างในเส้นทาง ข้อมูลรับรอง ส่วนท้าย และค่าคิวรีถูกแปลงอย่างปลอดภัย
