## 解析和重建 URL

当你需要快速检查完整 URL、逐项编辑其中某个部分，或在不手动处理分隔符的情况下重建干净链接时，这个工具会很有用。原始 URL 视图和结构化构建器会保持同步，因此你可以在两种思考方式之间切换而不丢失上下文。

## 每个部分代表什么

主机与凭据区域涵盖协议、可选凭据、主机名和端口。路径区域涵盖 pathname 和 fragment。查询参数以独立键值对的方式编辑，这对带有重复键、营销标签或 API 过滤条件的链接尤其有帮助。

## 适合哪些场景

- 调试回调 URL、API 端点和签名链接。
- 检查端口、片段或查询参数是否出现在预期的位置。
- 为文档、QA 用例和浏览器内工具重建链接，而无需离开当前页面。
