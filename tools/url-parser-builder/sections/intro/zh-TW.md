## 解析與重建 URL

當你需要快速檢查完整 URL、逐項編輯其中某個部分，或在不手動處理分隔符號的情況下重建乾淨連結時，這個工具會很有幫助。原始 URL 檢視和結構化建構器會保持同步，因此你可以在兩種思考方式之間切換而不失去上下文。

## 每個部分代表什麼

主機與憑證區域涵蓋協定、可選憑證、主機名稱和連接埠。路徑區域涵蓋 pathname 與 fragment。查詢參數以獨立鍵值對的方式編輯，對於包含重複鍵、行銷標籤或 API 篩選條件的連結特別實用。

## 適合哪些情境

- 偵錯回呼 URL、API 端點和簽署連結。
- 檢查連接埠、片段或查詢參數是否位於預期位置。
- 為文件、QA 測試資料和瀏覽器內工具重建連結，而不必離開目前頁面。
