## URL का विश्लेषण और पुनर्निर्माण

जब आपको पूरा URL जल्दी से जांचना हो, एक-एक भाग संपादित करना हो, या विभाजकों को हाथ से संभाले बिना साफ लिंक दोबारा बनाना हो, तब यह टूल उपयोगी है। कच्चा URL दृश्य और संरचित बिल्डर एक-दूसरे के साथ सिंक रहते हैं, इसलिए आप बिना संदर्भ खोए दोनों तरीकों के बीच जा सकते हैं।

## हर भाग का क्या अर्थ है

होस्ट और क्रेडेंशियल्स वाला भाग प्रोटोकॉल, वैकल्पिक क्रेडेंशियल्स, होस्टनेम और पोर्ट को कवर करता है। रूट वाला भाग pathname और fragment को कवर करता है। क्वेरी पैरामीटर अलग-अलग key-value जोड़ों के रूप में संपादित होते हैं, जो दोहराए गए keys, मार्केटिंग टैग या API filters वाले लिंक के लिए खास तौर पर उपयोगी है।

## यह कब उपयोगी है

- callback URL, API endpoint और signed links को debug करने के लिए।
- यह जांचने के लिए कि पोर्ट, fragment या query pair अपेक्षित जगह पर है या नहीं।
- documentation, QA fixtures और browser-based tools के लिए लिंक को पेज छोड़े बिना दोबारा बनाने के लिए।
