## URL を解析して再構築する

完全な URL をすばやく確認したいとき、1 つずつ要素を編集したいとき、区切り文字を手作業で扱わずにきれいなリンクを組み直したいときに、このツールが役立ちます。生の URL 表示と構造化ビルダーは同期されるため、2 つの見方を行き来しても文脈を失いません。

## 各パートの意味

ホストと認証情報のブロックには、プロトコル、任意の認証情報、ホスト名、ポートが含まれます。パスのブロックには pathname と fragment が含まれます。クエリパラメータは個別のキーと値の組として編集できるので、重複キー、マーケティングタグ、API フィルターを含むリンクで特に便利です。

## 役立つ場面

- コールバック URL、API エンドポイント、署名付きリンクをデバッグするとき。
- ポート、フラグメント、クエリ項目が期待どおりの位置にあるか確認するとき。
- ドキュメント、QA 用フィクスチャ、ブラウザー内ツール向けのリンクをページを離れずに組み直したいとき。
