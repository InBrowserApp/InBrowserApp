在瀏覽器中本地產生 KSUID，無需把目前這一批識別符傳送到其他服務。當你需要在分散式系統中保持低碰撞機率，同時又希望 ID 大致依建立時間排序，方便用於日誌、時間流、匯入資料或有序紀錄時，這個工具會很合適。

## 為什麼使用 KSUID

KSUID 會把 32 位時間戳和 128 位隨機資料組合起來，再編碼成 27 位 Base62 字串。這讓每個 ID 都足夠精簡、對 URL 友善、便於儲存，而內嵌的時間戳也讓較新的值通常排在較舊的值後面。

## 選擇目前時間或自訂時間

當你需要為正式資料、示範資料或日常批次產生建立新 ID 時，使用目前時間。切換到自訂時間則更適合可重現的測試夾具、補登資料、遷移範例，或需要呈現為某個特定時間建立的測試情境。

## 匯出前需要知道的事

KSUID 只保留秒級精度，因此任何毫秒輸入都會向下取整到該秒的起點。同一秒內產生的 ID 仍然具有唯一性，但它們最終的前後順序也會受到隨機載荷影響，所以更適合把 KSUID 理解為「可依時間排序」，而不是「嚴格連續遞增」。
