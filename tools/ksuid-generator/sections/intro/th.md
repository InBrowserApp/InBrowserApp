สร้าง KSUID ได้ภายในเบราว์เซอร์โดยไม่ต้องส่งชุดข้อมูลปัจจุบันไปยังบริการอื่น เครื่องมือนี้เหมาะเมื่อคุณต้องการตัวระบุที่ยังคงไม่ซ้ำกันในระบบแบบกระจาย และยังสามารถเรียงตามเวลาที่สร้างได้โดยประมาณสำหรับบันทึกเหตุการณ์ ฟีด งานนำเข้า หรือเรคอร์ดที่ต้องมีลำดับ

## ทำไมจึงควรใช้ KSUID

KSUID ผสานการประทับเวลา 32 บิตเข้ากับข้อมูลสุ่ม 128 บิต แล้วเข้ารหัสผลลัพธ์เป็นสตริง Base62 ความยาว 27 ตัวอักษร ทำให้แต่ละ ID มีขนาดกะทัดรัด ใช้งานกับ URL ได้ง่าย และจัดเก็บสะดวก ขณะเดียวกันการมี timestamp อยู่ภายในก็ช่วยให้ค่าที่ใหม่กว่ามักเรียงต่อจากค่าที่เก่ากว่า

## เลือกเวลาปัจจุบันหรือเวลากำหนดเอง

ใช้เวลาปัจจุบันเมื่อคุณต้องการ ID ใหม่สำหรับข้อมูล production เดโม หรือการสร้างแบบเป็นชุดตามปกติ สลับไปใช้ timestamp แบบกำหนดเองเมื่อคุณต้องการ fixture ที่ทำซ้ำได้ ระเบียนที่เติมย้อนหลัง ตัวอย่าง migration หรือ test case ที่ควรดูเหมือนถูกสร้างขึ้นในช่วงเวลาที่ระบุ

## สิ่งที่ควรรู้ก่อนส่งออก

KSUID เก็บความละเอียดเพียงระดับวินาที ดังนั้นอินพุตที่มีมิลลิวินาทีจะถูกปัดลงไปยังจุดเริ่มต้นของวินาทีนั้น ID ที่สร้างในวินาทีเดียวกันยังคงไม่ซ้ำกัน แต่ลำดับสุดท้ายก็ยังได้รับผลจากส่วนที่เป็นข้อมูลสุ่มด้วย ดังนั้นควรมอง KSUID ว่าเป็น ID ที่เรียงตามเวลาได้ ไม่ใช่ลำดับต่อเนื่องแบบเคร่งครัด
