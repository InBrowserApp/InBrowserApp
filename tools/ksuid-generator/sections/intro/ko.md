현재 배치를 다른 서비스로 보내지 않고 브라우저 안에서 KSUID를 생성합니다. 분산 시스템에서도 충돌 가능성을 낮추면서 생성 시점 기준으로 대략 정렬되는 식별자가 필요할 때 유용하며, 로그, 피드, 가져오기 데이터, 정렬된 레코드에 잘 맞습니다.

## 왜 KSUID를 사용하나요

KSUID는 32비트 타임스탬프와 128비트 무작위 데이터를 결합한 뒤 27자의 Base62 문자열로 인코딩합니다. 그래서 각 ID는 짧고 URL 친화적이며 저장하기 쉽고, 내장된 타임스탬프 덕분에 더 최근에 생성된 값이 보통 더 오래된 값 뒤에 정렬됩니다.

## 현재 시간과 사용자 지정 시간 선택

운영 데이터, 데모, 일반적인 일괄 생성에 새 ID가 필요하면 현재 시간을 사용하세요. 재현 가능한 fixture, 사후 적재 레코드, 마이그레이션 예시, 특정 시점에 생성된 것처럼 보여야 하는 테스트 케이스에는 사용자 지정 타임스탬프가 적합합니다.

## 내보내기 전에 알아둘 점

KSUID는 초 단위 정밀도만 유지하므로 밀리초가 포함된 입력은 해당 초의 시작 시점으로 내림됩니다. 같은 초에 만들어진 ID도 여전히 고유하지만, 최종 정렬 순서는 무작위 payload의 영향도 받습니다. 따라서 KSUID는 엄격한 순차 ID라기보다 시간 기준으로 정렬하기 쉬운 ID로 보는 편이 맞습니다.
