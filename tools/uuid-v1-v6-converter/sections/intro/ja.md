UUID v1 と UUID v6 には、タイムスタンプ、クロックシーケンス、ノード識別子という同じ中核情報が含まれています。UUID v1 はタイムスタンプを従来の UUID フィールド順で格納します。一方 UUID v6 は、それらのタイムスタンプビットを並べ替えることで、単純な辞書順ソートが作成時刻により自然に沿うようにします。

時間ベース UUID のレイアウトが異なるシステム間で識別子を移す必要があるときに、このツールを使用してください。UUID v1 を貼り付けると対応する UUID v6 を取得でき、UUID v6 を貼り付けると UUID v1 表現を復元できます。変換は決定的で、クロックシーケンスとノードバイトは変更されません。

## 使用する場面

- ID メタデータを保持したまま、レコードを従来の UUID v1 ストレージから UUID v6 へ移行する。
- UUID v1 と UUID v6 の値が混在するデータベース、ログ、キューをデバッグする。
- UUID v6 の値が、古い連携で期待される UUID v1 の値に戻せるか確認する。

## 入力形式

このコンバーターは、ハイフン付きの標準 UUID 文字列、コンパクトな 32 文字 UUID 文字列、大文字の UUID、`urn:uuid:` 値、波かっこで囲まれた UUID を受け付けます。結果は常に小文字の標準 UUID 形式に正規化されます。

## プライバシーと互換性に関する注意

UUID v1 と UUID v6 には、作成時刻やノード情報がエンコードされている場合があります。これらは秘密情報ではなく運用上の識別子として扱い、タイムスタンプやノードのメタデータが機密になり得る場合は公開を避けてください。このツールはブラウザー内でローカルに実行され、UUID はアップロードされません。
