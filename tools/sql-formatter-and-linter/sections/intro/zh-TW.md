## SQL 格式化與 Linter 是什麼？

SQL 格式化與 Linter 會在瀏覽器中重新排版 SQL 查詢，並同時檢查一小組高訊號問題。當你想要更整齊的查詢排版、穩定的關鍵字大小寫，以及對像 `SELECT *` 或缺少 `WHERE` 子句的 `UPDATE` 陳述式這類風險模式取得快速回饋時，它很有用。

## 何時使用

當你在審閱手寫 SQL、整理貼上的查詢準備分享，或比較不同 SQL 方言的格式時，可以使用這個工具。它很適合臨時查詢審閱、拉取請求整理，以及只在瀏覽器中完成格式化而不把 SQL 傳到伺服器的情境。

## 會檢查什麼

這個版本仍然把格式化器和 Linter 分開，但彼此協作。格式化使用具方言感知的版面選項執行 `sql-formatter`，而 Lint 則會顯示剖析錯誤、缺少分號、過於寬泛的 `SELECT *` 用法、不安全的變更、過長的行，以及關鍵字大小寫不一致等問題。
