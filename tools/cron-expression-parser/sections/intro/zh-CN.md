## 上线前先理解 Cron 调度

Cron 表达式很紧凑，但一个小小的字段错误就可能让任务运行频率远高于或远低于预期。这个解析器会在你的浏览器中验证表达式，用通俗语言解释调度，分解每个字段，并预览后续运行时间。

### 何时使用

- 在将部署、备份、清理或通知计划添加到服务器、CI 系统或任务运行器之前进行检查。
- 将复制来的 Cron 表达式与实际预期的调度进行对比。
- 通过一次只修改一个字段并观察说明更新，学习或调试 Cron 语法。

### 支持的格式

该工具支持标准五字段 Unix cron 表达式：分钟、小时、月中的日期、月份和星期。对于支持秒级精度的调度器，它也接受在开头包含秒字段的六字段表达式。

### 阅读结果

摘要会给出通俗描述，字段表显示原始表达式如何拆分。后续运行时间使用浏览器的本地时区，因此请将它们与实际运行任务的调度器所使用的时区进行对比。

### 注意事项

- 星期字段通常使用 `0` 或 `7` 表示星期日，也接受 `MON` 或 `FRI` 等名称。
- 使用 `JAN` 或 `DEC` 等月份名称，可以让生产调度更容易审阅。
- 如果你的调度器使用不同的 Cron 方言，请在该调度器自己的文档中确认 `?`、`L`、`W` 或 `#` 等特殊标记。
