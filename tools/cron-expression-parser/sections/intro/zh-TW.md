## 部署前先看懂 Cron 排程

Cron 表達式很精簡，但一個小小的欄位錯誤，就可能讓工作執行得比預期頻繁得多，或少得多。這個解析器會在你的瀏覽器中驗證表達式，以白話說明排程，分解每個欄位，並預覽即將到來的執行時間。

### 何時使用

- 將部署、備份、清理或通知排程加入伺服器、CI 系統或任務執行器之前，先檢查一次。
- 比對複製來的 cron 表達式是否符合你實際預期的排程。
- 透過一次修改一個欄位並觀察說明更新，學習或除錯 cron 語法。

### 支援格式

此工具支援標準五欄位 Unix cron 表達式：分鐘、小時、每月日期、月份與星期幾。它也接受前方帶秒數的六欄位表達式，適用於支援秒級精度的排程器。

### 閱讀結果

摘要會提供白話描述，而欄位表格則顯示原始表達式如何被拆分。即將執行的時間會使用你瀏覽器的本機時區，因此請將它們與實際執行工作的排程器所使用的時區進行比對。

### 注意事項

- 星期幾通常可用 `0` 或 `7` 表示星期日，也接受 `MON` 或 `FRI` 等名稱。
- `JAN` 或 `DEC` 等月份名稱可讓正式環境排程更容易檢視。
- 如果你的排程器使用不同的 cron 方言，請在該排程器自己的文件中確認 `?`、`L`、`W` 或 `#` 等特殊符號。
