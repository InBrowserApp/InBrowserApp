## Что такое Keccak?

Keccak — это семейство криптографических хеш-функций, которое служит основой для стандарта SHA-3 (Secure Hash Algorithm 3). Разработанный Гвидо Бертони, Жоаном Дэменом, Микаэлем Питерсом и Жилем Ван Ассшем, он выиграл конкурс хеш-функций NIST в 2012 году.

**Ключевые характеристики:**

- **Губчатая конструкция**: Использует инновативный дизайн губчатой функции с фазами поглощения и выжимания
- **Переменная длина вывода**: Может производить хеш-выводы любой желаемой длины
- **Высокий запас безопасности**: Разработан с существенными резервами безопасности
- **Отличается от SHA-1/SHA-2**: Основан на совершенно других математических принципах
- **Вариант Keccak[c=2d]**: Эта реализация использует оригинальную спецификацию Keccak с емкостью c = 2d (где d — длина вывода)

**Различия между Keccak и SHA-3 (FIPS 202):**
🔍 **Важное различие**: Оригинальный Keccak и стандартизированный SHA-3 **не идентичны**:

- **Оригинальный Keccak**: Использует емкость c = 2d и другое заполнение (заполнение Keccak: 0x01)
- **FIPS 202 SHA-3**: Использует емкость c = 2d, но другое заполнение (заполнение SHA-3: 0x06)
- **Разделение доменов**: Различие в заполнении обеспечивает, что Keccak и SHA-3 производят разные выводы для одного входа
- **Этот инструмент реализует**: **Оригинальную спецификацию Keccak** с параметризацией Keccak[c=2d]

**Статус безопасности:**
✅ **Keccak считается высокобезопасным** без известных практических атак. Он обеспечивает отличные запасы безопасности и устойчивость против различных криптоаналитических техник.

**Обычные применения:**

- Блокчейн Ethereum (использует оригинальный Keccak-256)
- Академические исследования и криптографические протоколы
- Приложения, требующие хеш-выводы переменной длины
- Системы, нуждающиеся в альтернативах семейству SHA-2
- Реализации блокчейна и криптовалют

**Преимущества перед традиционными хешами:**

- Принципиально другой дизайн снижает риск связанных атак
- Гибкая длина вывода (не ограничена фиксированными размерами)
- Сильная теоретическая база безопасности
- Устойчивость к атакам расширения длины
- Отличная производительность на различных платформах

**Техническая заметка:**

- **Keccak-256**: Производит 256-битный вывод (наиболее распространенный вариант)
- **Формула емкости**: c = 2d обеспечивает соответствующий уровень безопасности
- **Использование Ethereum**: Ethereum конкретно использует оригинальный Keccak-256, а не SHA3-256
