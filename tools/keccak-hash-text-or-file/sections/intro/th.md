## Keccak คืออะไร?

Keccak เป็นตระกูลของฟังก์ชันแฮชเข้ารหัสที่ทำหน้าที่เป็นพื้นฐานสำหรับมาตรฐาน SHA-3 (Secure Hash Algorithm 3) พัฒนาโดย Guido Bertoni, Joan Daemen, Michaël Peeters และ Gilles Van Assche ชนะการแข่งขันฟังก์ชันแฮช NIST ในปี 2012

**คุณลักษณะสำคัญ:**

- **โครงสร้างฟองน้ำ**: ใช้การออกแบบฟังก์ชันฟองน้ำที่เป็นนวัตกรรมใหม่ด้วยขั้นตอนการดูดซับและการบีบ
- **ความยาวเอาต์พุตแปรผัน**: สามารถสร้างเอาต์พุตแฮชที่มีความยาวตามต้องการ
- **ขอบเขตความปลอดภัยสูง**: ออกแบบมาพร้อมกับสำรองความปลอดภัยที่สำคัญ
- **แตกต่างจาก SHA-1/SHA-2**: อิงจากหลักการทางคณิตศาสตร์ที่แตกต่างโดยสิ้นเชิง
- **ตัวแปร Keccak[c=2d]**: การนำไปใช้นี้ใช้ข้อกำหนด Keccak ต้นฉบับด้วยความจุ c = 2d (โดยที่ d คือความยาวเอาต์พุต)

**ความแตกต่างระหว่าง Keccak และ SHA-3 (FIPS 202):**
🔍 **ความแตกต่างที่สำคัญ**: Keccak ต้นฉบับและ SHA-3 ที่มาตรฐาน **ไม่เหมือนกัน**:

- **Keccak ต้นฉบับ**: ใช้ความจุ c = 2d และการเติมที่แตกต่าง (การเติม Keccak: 0x01)
- **FIPS 202 SHA-3**: ใช้ความจุ c = 2d แต่การเติมที่แตกต่าง (การเติม SHA-3: 0x06)
- **การแยกโดเมน**: ความแตกต่างของการเติมทำให้ Keccak และ SHA-3 สร้างเอาต์พุตที่แตกต่างกันสำหรับอินพุตเดียวกัน
- **เครื่องมือนี้ใช้**: **ข้อกำหนด Keccak ต้นฉบับ** ด้วยการกำหนดพารามิเตอร์ Keccak[c=2d]

**สถานะความปลอดภัย:**
✅ **Keccak ถือว่าปลอดภัยสูง** โดยไม่มีการโจมตีที่ใช้งานได้จริงที่ทราบ มันให้ขอบเขตความปลอดภัยที่ยอดเยี่ยมและความต้านทานต่อเทคนิคการวิเคราะห์การเข้ารหัสต่างๆ

**การใช้งานทั่วไป:**

- บล็อกเชน Ethereum (ใช้ Keccak-256 ต้นฉบับ)
- การวิจัยทางวิชาการและโปรโตคอลการเข้ารหัส
- แอปพลิเคชันที่ต้องการเอาต์พุตแฮชที่มีความยาวแปรผัน
- ระบบที่ต้องการทางเลือกจากตระกูล SHA-2
- การใช้งานบล็อกเชนและสกุลเงินดิจิทัล

**ข้อดีเหนือแฮชแบบดั้งเดิม:**

- การออกแบบที่แตกต่างโดยพื้นฐานลดความเสี่ยงของการโจมตีที่เกี่ยวข้อง
- ความยาวเอาต์พุตที่ยืดหยุ่น (ไม่จำกัดเฉพาะขนาดคงที่)
- พื้นฐานความปลอดภัยเชิงทฤษฎีที่แข็งแกร่ง
- ความต้านทานต่อการโจมตีการขยายความยาว
- ประสิทธิภาพที่ยอดเยี่ยมบนแพลตฟอร์มต่างๆ

**หมายเหตุทางเทคนิค:**

- **Keccak-256**: สร้างเอาต์พุต 256-บิต (ตัวแปรที่พบบ่อยที่สุด)
- **สูตรความจุ**: c = 2d รับประกันระดับความปลอดภัยที่เหมาะสม
- **การใช้งาน Ethereum**: Ethereum ใช้โดยเฉพาะ Keccak-256 ต้นฉบับ ไม่ใช่ SHA3-256
