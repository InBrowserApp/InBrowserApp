## 什么是 NanoID？

NanoID 是一种小巧、URL 安全的唯一 ID 生成器，适用于现代 Web 应用、API 和内部工具。默认的 NanoID 使用 64 个字符的字母表生成 21 位字符串，在保持足够短小的同时，依然能提供大约 126 位随机性，适合放进 URL、文件名、测试数据或临时凭证里。

这个工具会在浏览器本地完成全部计算。你输入的自定义字符集和生成出的结果都不会离开当前页面，所以很适合快速做原型、批量准备 fixture、演示数据，或者一次性的运营支持工作。

**要点：**

- **URL 安全**：使用 A-Z、a-z、0-9、- 和 \_。
- **可自定义**：可根据约束调整长度与字符集。
- **安全随机**：在浏览器中使用加密随机数。
- **可导出为文本**：当前批次可以直接复制或下载，方便作为种子数据、演示内容或导入列表使用。

**实用建议：**

- 如果你需要通用且碰撞概率很低的标识符，优先保留默认的 21 位长度。
- 如果只是做临时 UI token、本地 mock 数据或短期演示，可以缩短长度，但长度越短、批量越大，碰撞风险就越高。
- 字符集越大，每个字符能承载的熵越多，因此通常可以在保持唯一性的同时把 ID 控制得更短。
- 自定义字符集时应确保每个字符都唯一。重复字符会破坏分布均匀性，所以这个工具会先拦截这类输入，再允许生成结果。
