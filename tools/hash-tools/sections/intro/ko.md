해시 도구 컬렉션은 마이그레이션된 해싱 유틸리티를 한곳에 모아 특정 도구를 열기 전에 올바른 알고리즘을 선택할 수 있게 합니다. 일상적인 파일 다이제스트, 레거시 호환성 검사, 키 기반 메시지 인증, Subresource Integrity 문자열, 비밀번호 해싱, 비밀번호 검증, 빠른 비암호화 체크섬을 다룹니다.

## When to use these tools

텍스트나 파일에 대해 반복 가능한 지문이 필요할 때, 예를 들어 다운로드한 아카이브를 게시된 SHA-256 체크섬과 비교할 때 암호화 다이제스트 도구를 사용하세요. 공유 비밀을 가진 누군가가 메시지를 만들었거나 승인했다는 것을 결과가 증명해야 할 때는 HMAC을 사용하세요. 원시 속도보다 구성 가능한 비용이 더 중요한 비밀번호 및 키 파생 워크플로에는 Argon2, bcrypt, PBKDF2 또는 scrypt를 사용하세요.

## Choosing safely

모든 해시가 보안에 적합한 것은 아닙니다. MD4, MD5, SHA-1은 레거시 시스템과 호환성 검사에는 유용하지만, 새 보안 민감 무결성 설계에는 사용하면 안 됩니다. CRC, Adler-32, MurmurHash, CityHash, xxHash는 빠른 체크섬 또는 버킷팅 해시이며, 변조 방지 서명이 아닙니다. 확실하지 않을 때는 공개 체크섬에는 SHA-256, 키 기반 검증에는 HMAC-SHA-256, 비밀번호 저장에는 Argon2id 또는 bcrypt를 선호하세요.

## Privacy and workflow

이 컬렉션의 개별 도구는 브라우저에서 실행됩니다. 선택한 도구가 공개 조회 동작을 명시적으로 문서화하지 않는 한 텍스트와 파일은 해당 도구에서 로컬로 처리되며, 해시 도구에는 그런 동작이 필요하지 않습니다. 민감한 자료의 경우 사용 후 생성된 값을 지우고, 공유되거나 녹화되는 브라우저 세션에 비밀 정보를 붙여넣지 마세요.
