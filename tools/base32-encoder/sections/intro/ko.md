## Base32란?

Base32는 OTP 비밀값, DNS-safe 토큰, 내보낸 설정 값 같은 바이너리 데이터를 텍스트 전용 채널이나 대소문자를 구분하지 않는 채널에서 다뤄야 할 때 유용합니다. 이것은 인코딩 계층이지 보안 계층이 아닙니다.

## 사용하면 좋은 때

- 텍스트 전용 채널로 보내기 전에 바이트, 텍스트, 파일을 Base32로 인코딩할 때.
- Base32 입력을 기대하는 시스템에 맞춰 OTP 비밀값, 내보낸 설정, 바이너리 데이터를 준비할 때.
- 원시 파일 바이트를 전송, 로그, 수동 입력에 쓰기 쉬운 문자열로 바꿀 때.

## 알아둘 점

- Base32는 Base64보다 데이터 크기를 더 많이 늘립니다.
- 원래 값을 암호화하거나 숨기지 않습니다.
- 어떤 시스템은 `=` 패딩을 요구하고, 어떤 시스템은 패딩 없는 출력을 허용하므로 수신 측 요구에 맞추는 편이 좋습니다.
