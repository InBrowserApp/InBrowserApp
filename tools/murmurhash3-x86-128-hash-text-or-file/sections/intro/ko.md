## MurmurHash3 (x86 128-bit)란?

MurmurHash3는 반복 가능하고 고르게 분포된 체크섬을 만들도록 설계된
빠른 비암호학적 해시 알고리즘입니다. x86 128-bit 변형은 보통 32개의
16진수 문자로 표시되는 16바이트 값을 반환하므로, 레코드, 파일 또는
캐시 키의 대규모 집합에 더 넓은 식별자가 필요할 때 32비트 해시보다
더 적합합니다.

**활용 분야:**

- **해시 테이블 및 샤딩**: 버킷, 파티션 또는 조회 테이블에 사용할
  안정적인 키를 만듭니다.
- **중복 제거**: 더 깊은 검사를 수행하기 전에 큰 텍스트 또는 파일
  집합을 간결한 128비트 지문으로 비교합니다.
- **캐시 키**: 빌드 산출물, 변환된 데이터 또는 생성된 콘텐츠에 대한
  결정론적 식별자를 생성합니다.
- **보안 목적이 아닌 무결성 검사**: 암호학적 보장이 필요하지 않을 때
  저장 또는 전송 중의 우발적 변경을 감지합니다.

**시드 동작:**

선택 사항인 시드는 32비트 부호 없는 값입니다. 다른 시스템과 결과를
맞춰야 할 때는 같은 시드를 사용하고, 특정 호환성 요구 사항이 없으면
`0`으로 두세요. 10진수 값과 `0x` 16진수 값을 사용할 수 있으며, 더 큰
값은 알고리즘에서 사용하는 동일한 32비트 범위로 래핑됩니다.

**안전 관련 참고 사항:**

MurmurHash3는 비밀번호 해싱, 서명 또는 변조 방지 검증 알고리즘이
아닙니다. 출력에 보안 속성이 필요하면 SHA-256, HMAC 또는 비밀번호
해싱 도구를 사용하세요. 이 도구는 공격 저항성보다 속도와 안정적인
분포가 더 중요한 로컬 오프라인 성능 중심 해싱에 가장 적합합니다.
