# CRC 체크섬 계산기

CRC(Cyclic Redundancy Check) 체크섬은 의도치 않은 데이터 변경을 감지하는
데 사용되는 간결한 값입니다. 빠른 오류 감지 값이 암호화 서명보다 더
유용한 네트워크 프레임, 아카이브 형식, 임베디드 프로토콜, 펌웨어
업데이트, 파일 무결성 워크플로에서 흔히 사용됩니다.

## 사용 시점

문서, 하드웨어 프로토콜, 파일 형식 또는 다른 시스템의 CRC 값을 비교해야
할 때 이 계산기를 사용하세요. 빠른 확인에는 텍스트를 붙여넣고, 정확한
바이트 스트림에서 체크섬을 계산해야 할 때는 파일을 가져오세요.

## 지원되는 변형

이 도구는 기존 InBrowser.App CRC 도구의 일반적인 변형을 계산합니다:
CRC-1, CRC-8, CRC-8 1-Wire, CRC-8 DVB-S2, CRC-16, CRC-16 CCITT,
CRC-16 Modbus, CRC-16 Kermit, CRC-16 XModem, CRC-24, CRC-32, CRC-32 MPEG-2,
CRCJAM 및 ECMA-182, GO-ISO, MS, NVME, REDIS, WE, XZ를 포함한 여러 CRC-64
모델.

## 주의할 점

CRC 변형 이름은 중요합니다. 같은 입력이라도 다항식, 초기값, 리플렉션
설정, 최종 XOR에 따라 서로 다른 값을 생성할 수 있습니다. 프로토콜이나
공급업체 사양과 맞추는 경우, 모든 CRC 폭을 서로 바꿔 쓸 수 있다고 보지
말고 해당 사양과 변형 이름이 일치하는 결과를 선택하세요.

CRC는 암호 저장, 서명 또는 변조 방지 보안이 아니라 의도치 않은 오류
감지를 위해 설계되었습니다. 보안에 민감한 검증에는 대신 암호화 해시 또는
서명 워크플로를 사용하세요.
