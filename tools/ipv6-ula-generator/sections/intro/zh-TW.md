## 什麼是 IPv6 唯一本地位址？

IPv6 唯一本地位址（ULA）用於站點、私有網路和互連組織內部的通訊。完整的 ULA 位址空間是 `fc00::/7`。其第八個位元是 **L 位元**：值為 `1` 時，選用本產生器使用的本地指派 `fd00::/8` 範圍；`fc00::/8` 的另一半則保留給其他指派方式。

ULA 預設不具全域可達性，但「本地」不表示保密，也不代表會自動安全。若營運人員設定相應路由，ULA 仍可跨越有路由的站點邊界、VPN 和私有互連。

## 此 RFC 4193 產生器如何建立 /48

此 RFC 4193 產生器會從 Web Crypto API 取得恰好 40 個隨機位元，並將它們與 `fd` 結合。結果是具統計唯一性的 48 位元站點前綴，例如 `fd12:3456:789a::/48`。產生作業全程留在瀏覽器中：不會收集 MAC 位址、時間戳、裝置識別碼或伺服器回應。

共有 `2^40` 個可能的 Global ID—約 1.1 兆個。安全隨機性可讓意外重複使用的機率極低，但無法保證兩個獨立產生的前綴永不衝突。請將選定的 `/48` 記錄在網路技術文件中，並持續一致地使用它。

## 規劃 65,536 個可用的 /64 子網路

在 `/48` 站點前綴之後，是 16 位元的 Subnet ID。從 `0000` 到 `ffff` 的值可提供 65,536 個可用的 `/64` 網路。例如，Subnet ID `00a0` 會將 `fd12:3456:789a::/48` 轉換為標準網路 `fd12:3456:789a:a0::/64`。

剩餘 64 位元是 Interface ID。此工具只規劃網路前綴；不會產生主機 `/128` 位址，也不會從 MAC 位址推導介面識別碼。

## ULA 適合用在哪裡，又不適合用在哪裡

ULA 適合穩定的內部定址、透過 VPN 連線的站點、實驗室網路，以及在使用全域單播 IPv6 時仍應保有內部前綴的服務。ULA 本身不是防火牆，也不是固有的安全邊界。請套用一般存取控制，在站點邊界過濾不當的 ULA 流量，並避免將內部 ULA 記錄放入公用 DNS。

主機可同時使用 ULA 和全域單播位址。需要網際網路可達性時使用全域位址，而需要穩定私有路徑時，則使用可長期沿用的 ULA 前綴。
