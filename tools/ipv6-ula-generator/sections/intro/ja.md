## IPv6 ユニークローカルアドレスとは？

IPv6 ユニークローカルアドレス（ULA）は、サイト、プライベートネットワーク、相互接続された組織内の通信を目的としています。ULA 全体のアドレス空間は `fc00::/7` です。8 番目のビットは **L ビット**です。値 `1` は、この生成ツールが使用するローカル割り当ての `fd00::/8` 範囲を選択し、`fc00::/8` 側の半分は別の割り当て方式用に予約されています。

ULA はデフォルトではグローバルに到達できませんが、「ローカル」は秘密であることや自動的な安全性を意味しません。運用者が経路を設定すれば、ルーティングされたサイト境界、VPN、プライベート相互接続を越えて通信できます。

## この RFC 4193 生成ツールが /48 を構築する仕組み

この RFC 4193 生成ツールは Web Crypto API からちょうど 40 ビットの乱数を取得し、`fd` と組み合わせます。結果は `fd12:3456:789a::/48` のような、統計的に一意の 48 ビットサイトプレフィックスになります。生成はブラウザー内で完結し、MAC アドレス、タイムスタンプ、デバイス識別子、サーバーレスポンスを収集しません。

グローバル ID の候補は `2^40` 個、約 1.1 兆個あります。安全な乱数により偶発的な再利用の可能性は低くなりますが、独立に生成された 2 つのプレフィックスが決して衝突しないとは保証できません。選択した `/48` をネットワーク資料に記録し、一貫して再利用してください。

## 利用可能な 65,536 個の /64 サブネットを計画する

`/48` サイトプレフィックスの後には 16 ビットのサブネット ID が続きます。`0000` から `ffff` までの値により、65,536 個の `/64` ネットワークを利用できます。たとえば、サブネット ID `00a0` を指定すると、`fd12:3456:789a::/48` は正規ネットワーク `fd12:3456:789a:a0::/64` になります。

残りの 64 ビットはインターフェース ID です。このツールが計画するのはネットワークプレフィックスのみです。ホストの `/128` アドレスを生成したり、MAC アドレスからインターフェース識別子を導出したりはしません。

## ULA が適する場所と適さない場所

ULA は、安定した内部アドレス設計、VPN で接続されたサイト、ラボネットワーク、およびグローバルユニキャスト IPv6 を併用しながら内部プレフィックスを維持したいサービスに適しています。ULA はファイアウォールではなく、それ自体がセキュリティ境界になるわけでもありません。通常のアクセス制御を適用し、サイト境界で不適切な ULA トラフィックをフィルタリングし、内部 ULA レコードをパブリック DNS に登録しないでください。

ホストは ULA とグローバルユニキャストアドレスを同時に使用できます。インターネット到達性にはグローバルアドレスを、必要なプライベート経路には継続して使用する ULA プレフィックスを使ってください。
