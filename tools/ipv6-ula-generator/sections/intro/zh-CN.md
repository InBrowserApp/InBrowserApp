## 什么是 IPv6 唯一本地地址？

IPv6 唯一本地地址（ULA）用于站点、专用网络和互联组织内部的通信。完整的 ULA 地址空间是 `fc00::/7`。其第八位是 **L 位**：值 `1` 选择此生成器使用的本地分配 `fd00::/8` 范围，而 `fc00::/8` 的另一半仍保留给其他分配方式。

ULA 默认不具备全球可达性，但“本地”并不代表保密或自动安全。只要网络运营人员配置了相应路径，它们就能跨越已路由的站点边界、VPN 和专用互连。

## 此 RFC 4193 生成器如何构建 /48

此 RFC 4193 生成器向 Web Crypto API 请求恰好 40 个随机位，并将它们与 `fd` 组合。结果是一个在统计意义上唯一的 48 位站点前缀，例如 `fd12:3456:789a::/48`。生成过程始终在浏览器中完成：它不会收集 MAC 地址、时间戳、设备标识符或服务器响应。

全局 ID 共有 `2^40` 种可能，约为 1.1 万亿种。安全随机数使意外重复的可能性很低，但无法保证两个独立生成的前缀永远不会冲突。请把选定的 `/48` 记录在网络文档中，并始终复用它。

## 规划 65,536 个可用的 /64 子网

在 `/48` 站点前缀之后是 16 位子网 ID。`0000` 至 `ffff` 的取值可提供 65,536 个 `/64` 网络。例如，子网 ID `00a0` 会把 `fd12:3456:789a::/48` 转换为标准网络 `fd12:3456:789a:a0::/64`。

剩余的 64 位是接口 ID。此工具只规划网络前缀；它不会生成主机 `/128` 地址，也不会根据 MAC 地址派生接口标识符。

## ULA 适用于哪些场景，又不适用于哪些场景

ULA 非常适合稳定的内部寻址、通过 VPN 连接的站点、实验室网络，以及那些在使用全球单播 IPv6 的同时仍需保留内部前缀的服务。它们不是防火墙，也不是天然的安全边界。请应用常规访问控制，在站点边界过滤不合适的 ULA 流量，并避免把内部 ULA 记录发布到公共 DNS。

一台主机可以同时使用 ULA 和全球单播地址。请使用全球地址访问互联网，并把持久的 ULA 前缀用于需要它的专用路径。
