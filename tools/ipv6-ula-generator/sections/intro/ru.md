## Что такое уникальный локальный IPv6-адрес?

Уникальный локальный IPv6-адрес (ULA) предназначен для обмена данными внутри сайтов, частных сетей и связанных организаций. Полное пространство ULA — `fc00::/7`. Его восьмой бит — **бит L**: значение `1` выбирает локально назначенный диапазон `fd00::/8`, используемый этим генератором, а половина `fc00::/8` остаётся зарезервированной для другого метода назначения.

По умолчанию ULA-адреса не имеют глобальной доступности, но «локальный» не означает секретный или автоматически защищённый. Они могут пересекать маршрутизируемые границы сайтов, VPN и частные межсетевые соединения, если операторы настроят соответствующие маршруты.

## Как этот генератор RFC 4193 создаёт /48

Этот генератор RFC 4193 запрашивает у Web Crypto API ровно 40 случайных битов и объединяет их с `fd`. В результате получается статистически уникальный 48-битный префикс сайта, например `fd12:3456:789a::/48`. Генерация выполняется в браузере: она не собирает MAC-адрес, метку времени, идентификатор устройства или ответ сервера.

Существует `2^40` возможных глобальных ID — около 1,1 триллиона. Криптографически стойкая случайность снижает вероятность случайного повторного использования, но не может гарантировать, что два независимо сгенерированных префикса никогда не совпадут. Зафиксируйте выбранный `/48` в сетевой документации и в дальнейшем используйте его постоянно.

## Планирование 65 536 доступных подсетей /64

За префиксом сайта `/48` следует 16-битный ID подсети. Значения от `0000` до `ffff` дают 65 536 возможных сетей `/64`. Например, ID подсети `00a0` преобразует `fd12:3456:789a::/48` в каноническую сеть `fd12:3456:789a:a0::/64`.

Оставшиеся 64 бита — это ID интерфейса. Этот инструмент планирует только сетевые префиксы; он не создаёт адреса хостов `/128` и не формирует идентификаторы интерфейсов на основе MAC-адресов.

## Где следует использовать ULA, а где — нет

ULA-адреса подходят для стабильной внутренней адресации, сайтов, соединённых через VPN, лабораторных сетей и сервисов, которым необходимо сохранять внутренний префикс, одновременно используя глобальные одноадресные IPv6-адреса. ULA-адреса не являются межсетевым экраном или неотъемлемой границей безопасности. Применяйте обычные средства контроля доступа, фильтруйте недопустимый ULA-трафик на границах сайтов и не публикуйте внутренние записи ULA в общедоступной системе DNS.

Хост может одновременно использовать адрес ULA и глобальный одноадресный адрес. Используйте глобальный адрес для связи с Интернетом, а постоянный префикс ULA — для частных соединений, которым он необходим.
