## IPv6 고유 로컬 주소란 무엇인가요?

IPv6 고유 로컬 주소(ULA)는 사이트, 사설 네트워크 및 연결된 조직 내부의 통신을 위한 주소입니다. 전체 ULA 주소 공간은 `fc00::/7`입니다. 여덟 번째 비트는 **L 비트**로, 값 `1`은 이 생성기가 사용하는 로컬 할당 `fd00::/8` 범위를 선택하며 `fc00::/8` 절반은 다른 할당 방식을 위해 예약되어 있습니다.

ULA는 기본적으로 전역에서 도달할 수 없지만, ‘로컬’이라고 해서 비밀이거나 자동으로 안전한 것은 아닙니다. 운영자가 해당 경로를 구성하면 라우팅된 사이트 경계, VPN, 사설 상호 연결을 통과할 수 있습니다.

## 이 RFC 4193 생성기가 /48을 만드는 방법

이 RFC 4193 생성기는 Web Crypto API에 정확히 40비트 난수를 요청해 `fd`와 결합합니다. 결과는 `fd12:3456:789a::/48`과 같은 통계적으로 고유한 48비트 사이트 프리픽스입니다. 생성은 브라우저 안에서만 이루어지며 MAC 주소, 타임스탬프, 기기 식별자 또는 서버 응답을 수집하지 않습니다.

가능한 글로벌 ID는 `2^40`개로, 약 1조 1천억 개입니다. 보안 난수를 사용하면 우연한 중복 가능성이 낮아지지만, 독립적으로 생성된 두 프리픽스가 절대 충돌하지 않는다고 보장할 수는 없습니다. 선택한 `/48`을 네트워크 문서에 기록하고 일관되게 재사용하세요.

## 사용 가능한 65,536개의 /64 서브넷 계획하기

사이트 프리픽스 `/48` 다음에는 16비트 서브넷 ID가 옵니다. `0000`부터 `ffff`까지의 값으로 65,536개의 `/64` 네트워크를 만들 수 있습니다. 예를 들어 서브넷 ID `00a0`을 사용하면 `fd12:3456:789a::/48`이 정규 네트워크 `fd12:3456:789a:a0::/64`로 바뀝니다.

나머지 64비트는 인터페이스 ID입니다. 이 도구는 네트워크 프리픽스만 계획하며, 호스트 `/128` 주소를 생성하거나 MAC 주소에서 인터페이스 식별자를 파생하지 않습니다.

## ULA를 사용해야 하는 곳과 사용하지 말아야 하는 곳

ULA는 안정적인 내부 주소 지정, VPN으로 연결된 사이트, 랩 네트워크, 전역 유니캐스트 IPv6를 함께 사용하면서 내부 프리픽스를 유지해야 하는 서비스에 적합합니다. ULA 자체는 방화벽이나 본질적인 보안 경계가 아닙니다. 일반적인 액세스 제어를 적용하고, 사이트 경계에서 부적절한 ULA 트래픽을 필터링하며, 내부 ULA 레코드를 공용 DNS에 게시하지 마세요.

호스트는 ULA와 전역 유니캐스트 주소를 동시에 사용할 수 있습니다. 인터넷 통신에는 전역 주소를 사용하고, 필요한 사설 경로에는 지속적으로 사용할 ULA 프리픽스를 사용하세요.
