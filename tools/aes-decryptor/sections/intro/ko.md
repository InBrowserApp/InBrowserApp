AES 복호화는 같은 AES 키 자료로 암호화된 데이터에서 평문을 복원합니다. 이 도구는 InBrowser.App AES Encryptor가 생성한 JSON 엔벌로프에 맞게 설계되었습니다. 엔벌로프는 알고리즘, 키 파생 설정, salt, IV, 암호문, 평문 메타데이터를 함께 보관하며, 비밀번호나 원시 키는 별도로 유지합니다.

모든 작업은 브라우저 Web Crypto API를 사용해 로컬에서 이루어집니다. 암호화된 JSON, 비밀번호, 원시 키, 복호화 결과는 업로드되지 않습니다.

## 이 도구를 사용할 때

누군가 `inbrowser-aes-v1` JSON 엔벌로프를 제공했거나, 이전에 일치하는 AES Encryptor 페이지로 암호화한 노트, 토큰, 구성 스니펫 또는 파일을 복구해야 할 때 사용하세요.

엔벌로프가 비밀번호로 만들어졌다면 같은 비밀번호를 입력하세요. 그러면 도구가 저장된 PBKDF2 해시, 반복 횟수, salt, AES 모드, 키 길이를 다시 사용합니다. 엔벌로프가 원시 키로 만들어졌다면 엔벌로프에 기록된 정확한 길이의 16진수 키를 붙여넣으세요.

## 실용적인 참고 사항

AES-GCM은 암호화된 데이터를 인증하므로 잘못된 키나 변경된 JSON은 변조된 평문을 반환하는 대신 실패해야 합니다. AES-CBC와 AES-CTR은 호환되는 엔벌로프를 복호화할 수 있지만, 자체적으로 암호문을 인증하지는 않습니다.

비밀번호나 원시 키는 JSON 엔벌로프와 별도로 보관하세요. 엔벌로프와 키 자료를 모두 가진 사람은 평문을 복구할 수 있습니다. 파일 엔벌로프의 경우 복구된 다운로드는 JSON에 저장된 원본 파일 이름과 미디어 타입을 사용합니다.
