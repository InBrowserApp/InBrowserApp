AES 解密会使用相同的 AES 密钥材料，从加密数据中还原明文。此工具专为 InBrowser.App AES 加密器生成的 JSON 信封而设计。信封会把算法、密钥派生设置、盐、IV、密文和明文元数据保存在一起，而密码或原始密钥会单独保留。

所有处理都通过浏览器 Web Crypto API 在本地完成。加密 JSON、密码、原始密钥和解密结果都不会上传。

## 何时使用此工具

当有人给你一个 `inbrowser-aes-v1` JSON 信封，或者你需要还原之前用匹配的 AES 加密器页面加密的笔记、令牌、配置片段或文件时，可以使用此工具。

如果信封是使用密码创建的，请输入同一个密码，工具会复用已存储的 PBKDF2 哈希、迭代次数、盐、AES 模式和密钥长度。如果信封是使用原始密钥创建的，请粘贴信封中记录的精确长度的十六进制密钥。

## 实用说明

AES-GCM 会认证加密数据，因此密钥错误或 JSON 被更改时应当失败，而不是返回被篡改的明文。AES-CBC 和 AES-CTR 可以解密兼容信封，但它们本身不会认证密文。

请将密码或原始密钥与 JSON 信封分开保存。任何同时拥有信封和密钥材料的人都可以恢复明文。对于文件信封，还原后的下载会使用 JSON 中存储的原始文件名和媒体类型。
