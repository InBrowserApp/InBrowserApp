AES 解密會使用相同的 AES 金鑰材料，從已加密資料還原明文。這個工具專為 InBrowser.App AES 加密器產生的 JSON 封套設計。封套會將演算法、金鑰衍生設定、salt、IV、密文與明文中繼資料保存在一起，而密碼或原始金鑰則保持分離。

所有作業都會透過瀏覽器 Web Crypto API 在本機完成。已加密的 JSON、密碼、原始金鑰與解密結果都不會上傳。

## 何時使用此工具

當有人提供你 `inbrowser-aes-v1` JSON 封套，或你需要復原先前用相符 AES 加密器頁面加密的筆記、權杖、設定片段或檔案時，可以使用此工具。

如果封套是使用密碼建立，請輸入相同密碼，工具會重用已儲存的 PBKDF2 雜湊、迭代次數、salt、AES 模式與金鑰長度。如果封套是使用原始金鑰建立，請貼上封套中記錄的確切長度十六進位金鑰。

## 實務注意事項

AES-GCM 會驗證已加密資料，因此錯誤金鑰或遭變更的 JSON 應該會失敗，而不是回傳被改動的明文。AES-CBC 和 AES-CTR 可以解密相容封套，但它們本身不會驗證密文。

請將密碼或原始金鑰與 JSON 封套分開保存。任何同時擁有封套和金鑰材料的人都能復原明文。對於檔案封套，復原後的下載會使用 JSON 中儲存的原始檔名與媒體類型。
