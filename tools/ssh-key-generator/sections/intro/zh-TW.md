## 什麼是 SSH 金鑰對？

SSH 金鑰對是用於向伺服器、Git 主機、部署系統和其他 SSH 型服務驗證身分的一組公鑰和私鑰。公鑰可以分享。私鑰必須保密。

這個產生器會完全在你的瀏覽器中建立 OpenSSH 格式的 Ed25519 或 RSA 金鑰。它也會顯示 SHA-256 指紋，這是你驗證金鑰時 OpenSSH 通常顯示的精簡值。

## 何時使用這個工具

- 為測試伺服器、Git remote、容器或臨時實驗環境建立開發用金鑰。
- 當你需要為新的 SSH 存取使用現代且精簡的預設選項時，產生 Ed25519 金鑰。
- 當較舊的服務不支援 Ed25519 時，產生 RSA 金鑰。
- 將公鑰複製到 `authorized_keys`，同時讓私鑰留在你的裝置上。

## 如何選擇演算法

Ed25519 是大多數新 SSH 金鑰的最佳預設選擇，因為它體積小、速度快，且目前的 OpenSSH 版本廣泛支援。RSA 適合用於與較舊設備、舊版 Git 伺服器，或仍要求 RSA 金鑰的政策需求相容。

對 RSA 來說，4096 位元是保守的預設值。較小的 2048 位元金鑰速度較快且仍很常見，但許多團隊現在偏好為新的長期金鑰使用 3072 或 4096 位元。

## 注意事項

- 這裡產生的私鑰未加密。如果需要通關密語，請使用 `ssh-keygen -p -f <key-file>` 加上。
- 使用限制性權限儲存私鑰，例如 `chmod 600 <key-file>`。
- 不要將私鑰貼到工單、聊天、記錄檔或不明網頁中。
- 當筆記型電腦、CI secret，或包含私鑰的備份可能外洩時，請輪替金鑰。
