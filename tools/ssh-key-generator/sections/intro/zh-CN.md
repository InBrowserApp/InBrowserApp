## 什么是 SSH 密钥对？

SSH 密钥对由公钥和私钥组成，用于向服务器、Git 托管平台、部署系统和其他基于 SSH 的服务进行身份验证。公钥可以分享，私钥必须保密。

此生成器会完全在你的浏览器中创建 OpenSSH 格式的 Ed25519 或 RSA 密钥。它还会显示 SHA-256 指纹，这是 OpenSSH 在你验证密钥时常见显示的紧凑值。

## 何时使用此工具

- 为测试服务器、Git 远程仓库、容器或临时实验环境创建开发密钥。
- 当你需要为新的 SSH 访问使用现代、紧凑的默认值时，生成 Ed25519 密钥。
- 当较旧的服务不支持 Ed25519 时，生成 RSA 密钥。
- 将公钥复制到 `authorized_keys`，同时将私钥保留在你的设备上。

## 如何选择算法

Ed25519 是大多数新 SSH 密钥的最佳默认选择，因为它体积小、速度快，并且受到当前 OpenSSH 版本的广泛支持。RSA 适用于需要兼容较旧设备、旧版 Git 服务器，或仍要求 RSA 密钥的策略要求。

对于 RSA，4096 位是较保守的默认值。较小的 2048 位密钥速度更快，也仍然常见，但许多团队现在更偏好为新的长期密钥使用 3072 或 4096 位。

## 需要注意什么

- 此处生成的私钥未加密。如果需要密码短语，请使用 `ssh-keygen -p -f <key-file>` 添加。
- 使用严格的权限保存私钥，例如 `chmod 600 <key-file>`。
- 不要将私钥粘贴到工单、聊天、日志或未知网页中。
- 当笔记本电脑、CI 密钥或包含私钥的备份可能暴露时，请轮换密钥。
