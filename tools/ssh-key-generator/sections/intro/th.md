## คู่คีย์ SSH คืออะไร?

คู่คีย์ SSH คือคีย์สาธารณะและคีย์ส่วนตัวที่ใช้ยืนยันตัวตนกับเซิร์ฟเวอร์ โฮสต์ Git ระบบปรับใช้ และบริการอื่นที่ใช้ SSH คีย์สาธารณะแชร์ได้ ส่วนคีย์ส่วนตัวต้องเก็บเป็นความลับ

เครื่องมือนี้สร้างคีย์ Ed25519 หรือ RSA ในรูปแบบ OpenSSH ทั้งหมดภายในเบราว์เซอร์ของคุณ และยังแสดงลายนิ้วมือ SHA-256 ซึ่งเป็นค่าขนาดกะทัดรัดที่ OpenSSH มักแสดงเมื่อคุณตรวจสอบคีย์

## เมื่อใดควรใช้เครื่องมือนี้

- สร้างคีย์สำหรับงานพัฒนาสำหรับเซิร์ฟเวอร์ทดสอบ, Git remote, คอนเทนเนอร์ หรือสภาพแวดล้อมแล็บชั่วคราว
- สร้างคีย์ Ed25519 เมื่อคุณต้องการค่าเริ่มต้นที่ทันสมัยและกะทัดรัดสำหรับการเข้าถึง SSH ใหม่
- สร้างคีย์ RSA เมื่อบริการรุ่นเก่าไม่รองรับ Ed25519
- คัดลอกคีย์สาธารณะไปยัง `authorized_keys` โดยเก็บคีย์ส่วนตัวไว้บนอุปกรณ์ของคุณ

## วิธีเลือกอัลกอริทึม

Ed25519 เป็นค่าเริ่มต้นที่ดีที่สุดสำหรับคีย์ SSH ใหม่ส่วนใหญ่ เพราะมีขนาดเล็ก รวดเร็ว และได้รับการรองรับอย่างกว้างขวางจาก OpenSSH เวอร์ชันปัจจุบัน RSA มีประโยชน์สำหรับความเข้ากันได้กับอุปกรณ์รุ่นเก่า เซิร์ฟเวอร์ Git แบบเดิม หรือข้อกำหนดนโยบายที่ยังคาดหวังคีย์ RSA

สำหรับ RSA ค่า 4096 บิตเป็นค่าเริ่มต้นที่รัดกุม คีย์ขนาดเล็กกว่า 2048 บิตสร้างได้เร็วกว่าและยังพบใช้ทั่วไป แต่ตอนนี้หลายทีมเลือก 3072 หรือ 4096 บิตสำหรับคีย์ใหม่ที่ใช้งานระยะยาว

## สิ่งที่ควรคำนึงถึง

- คีย์ส่วนตัวที่สร้างที่นี่ไม่ได้เข้ารหัส เพิ่มรหัสผ่านด้วย `ssh-keygen -p -f <key-file>` หากคุณต้องการ
- จัดเก็บคีย์ส่วนตัวด้วยสิทธิ์ที่จำกัด เช่น `chmod 600 <key-file>`
- อย่าวางคีย์ส่วนตัวลงในทิกเก็ต แชต ล็อก หรือหน้าเว็บที่ไม่รู้จัก
- เปลี่ยนคีย์ใหม่เมื่อแล็ปท็อป, CI secret, หรือข้อมูลสำรองที่มีคีย์ส่วนตัวอาจรั่วไหล
