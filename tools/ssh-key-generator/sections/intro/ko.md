## SSH 키 쌍이란 무엇인가요?

SSH 키 쌍은 서버, Git 호스트, 배포 시스템 및 기타 SSH 기반 서비스에 인증하는 데 사용하는 공개 키와 개인 키입니다. 공개 키는 공유할 수 있습니다. 개인 키는 반드시 비밀로 유지해야 합니다.

이 생성기는 OpenSSH 형식의 Ed25519 또는 RSA 키를 전적으로 브라우저 안에서 만듭니다. 또한 키를 확인할 때 OpenSSH가 일반적으로 표시하는 간결한 값인 SHA-256 지문도 보여 줍니다.

## 이 도구를 사용할 때

- 테스트 서버, Git 원격 저장소, 컨테이너 또는 임시 실습 환경용 개발 키를 만듭니다.
- 새 SSH 접근에 사용할 현대적이고 작은 기본값이 필요할 때 Ed25519 키를 생성합니다.
- 오래된 서비스가 Ed25519를 지원하지 않을 때 RSA 키를 생성합니다.
- 개인 키는 기기에 보관한 채 공개 키를 `authorized_keys`에 복사합니다.

## 알고리즘을 선택하는 방법

Ed25519는 작고 빠르며 최신 OpenSSH 버전에서 널리 지원되므로 대부분의 새 SSH 키에 가장 좋은 기본값입니다. RSA는 오래된 장비, 레거시 Git 서버 또는 여전히 RSA 키를 요구하는 정책과의 호환성에 유용합니다.

RSA의 경우 4096비트는 보수적인 기본값입니다. 더 작은 2048비트 키는 더 빠르고 여전히 흔히 쓰이지만, 많은 팀은 이제 새 장기 사용 키에 3072비트 또는 4096비트를 선호합니다.

## 유의할 점

- 여기에서 생성된 개인 키는 암호화되어 있지 않습니다. 패스프레이즈가 필요하면 `ssh-keygen -p -f <key-file>`로 추가하세요.
- 개인 키는 `chmod 600 <key-file>` 같은 제한적인 권한으로 저장하세요.
- 개인 키를 티켓, 채팅, 로그 또는 알 수 없는 웹 페이지에 붙여넣지 마세요.
- 개인 키가 들어 있는 노트북, CI 시크릿 또는 백업이 노출되었을 가능성이 있으면 키를 교체하세요.
