DNS 查询会检查某个域名返回的公共 DNS 记录。当你在验证新网站上线、调试邮件投递、检查 CDN 或负载均衡器变更，或确认 DNSSEC 相关响应在不同解析器之间是否有差异时，它会很有用。

## 何时使用

当你需要在浏览器端快速查看常见 DNS 记录类型时，可以使用这个工具。A 和 AAAA 记录显示 IPv4 和 IPv6 目标地址，CNAME 记录显示别名，MX 记录标识邮件交换器，TXT 记录通常包含 SPF 或验证令牌，而 NS/SOA/CAA/SRV/HTTPS/SVCB 记录会展示委派、权威、证书、服务和现代端点提示。

## 工作原理

查询会在你的浏览器中通过 DNS over HTTPS 运行。选择一个解析器，选择一个或多个记录类型，然后提交域名或 URL。URL 会在发送查询前规范化为主机名，因此粘贴 `https://www.example.com/path` 会查询 `www.example.com`。

## 解读结果

每种记录类型都会单独显示 DNS 响应代码、解析器标志、答案行和原始 JSON。`NoError` 表示 DNS 服务器已成功响应，但对于某个具体类型，它仍然可能不返回任何答案行。`NXDomain`、`ServFail` 或 `Refused` 通常表示名称不存在、解析器无法完成查询，或解析器策略阻止了请求。

## 隐私与限制

查询会发送到所选的 DNS over HTTPS 解析器，而不是 InBrowser.App 服务器。解析器行为、缓存状态、DNSSEC 验证和本地网络过滤都可能影响结果。这个工具不能替代从多个网络运行权威 `dig` 检查，但它可以快速查看公共 DoH 解析器从你当前浏览器返回的内容。
