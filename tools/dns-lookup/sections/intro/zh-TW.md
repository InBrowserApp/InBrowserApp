DNS 查詢會檢查針對網域名稱傳回的公開 DNS 記錄。當你正在驗證新網站上線、偵錯電子郵件遞送、檢查 CDN 或負載平衡器變更，或確認不同解析器中的 DNSSEC 相關回應是否不同時，這個工具很有用。

## 使用時機

當你需要在瀏覽器端快速取得常見 DNS 記錄類型的答案時，使用這個工具。A 和 AAAA 記錄會顯示 IPv4 和 IPv6 目的地，CNAME 記錄會顯示別名，MX 記錄會識別郵件交換器，TXT 記錄通常包含 SPF 或驗證權杖，而 NS/SOA/CAA/SRV/HTTPS/SVCB 記錄會揭示委派、權威、憑證、服務和現代端點提示。

## 運作方式

查詢會在你的瀏覽器中透過 DNS over HTTPS 執行。選擇一個解析器、選取一種或多種記錄類型，然後送出網域或 URL。URL 會在送出查詢前正規化為其主機名稱，因此貼上 `https://www.example.com/path` 會查詢 `www.example.com`。

## 閱讀結果

每種記錄類型會分別顯示，包含 DNS 回應碼、解析器旗標、答案列和原始 JSON。`NoError` 表示 DNS 伺服器已成功回答，但特定類型仍可能沒有任何答案列。`NXDomain`、`ServFail` 或 `Refused` 通常表示名稱不存在、解析器無法完成查詢，或解析器政策封鎖了請求。

## 隱私與限制

查詢會傳送至選取的 DNS over HTTPS 解析器，而不是 InBrowser.App 伺服器。解析器行為、快取狀態、DNSSEC 驗證和本機網路篩選都可能影響結果。此工具無法取代從多個網路進行的權威 `dig` 檢查，但它能讓你快速檢視公開 DoH 解析器從目前瀏覽器傳回的內容。
