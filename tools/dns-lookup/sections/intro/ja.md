DNS ルックアップは、ドメイン名に対して返される公開 DNS レコードを確認します。新しいサイト公開の検証、メール配信のデバッグ、CDN やロードバランサー変更の確認、または DNSSEC 関連のレスポンスがリゾルバー間で異なるかどうかを確かめるときに役立ちます。

## 使う場面

一般的な DNS レコードタイプについて、ブラウザー上ですばやく答えを得たいときにこのツールを使用します。A と AAAA レコードは IPv4 と IPv6 の宛先を示し、CNAME レコードはエイリアスを示します。MX レコードはメール交換サーバーを識別し、TXT レコードには SPF や検証トークンが含まれることがよくあります。NS/SOA/CAA/SRV/HTTPS/SVCB レコードは、委任、権威情報、証明書、サービス、最新のエンドポイントヒントを明らかにします。

## 仕組み

ルックアップは DNS over HTTPS を使ってブラウザー内で実行されます。リゾルバーを選び、1 つ以上のレコードタイプを選択して、ドメインまたは URL を送信します。URL はクエリ送信前にホスト名へ正規化されるため、`https://www.example.com/path` を貼り付けると `www.example.com` が照会されます。

## 結果の読み方

各レコードタイプは、DNS レスポンスコード、リゾルバーフラグ、回答行、生の JSON とともに個別に表示されます。`NoError` は DNS サーバーが正常に応答したことを意味しますが、特定のタイプについて回答行がない場合もあります。`NXDomain`、`ServFail`、`Refused` は通常、名前が存在しない、リゾルバーがルックアップを完了できなかった、またはリゾルバーのポリシーによってリクエストがブロックされたことを意味します。

## プライバシーと制限

クエリは InBrowser.App サーバーではなく、選択した DNS over HTTPS リゾルバーへ送信されます。リゾルバーの動作、キャッシュ状態、DNSSEC 検証、ローカルネットワークのフィルタリングはいずれも結果に影響する可能性があります。このツールは複数ネットワークからの権威 `dig` チェックを置き換えるものではありませんが、現在のブラウザーから公開 DoH リゾルバーが返す内容をすばやく確認する方法として有用です。
