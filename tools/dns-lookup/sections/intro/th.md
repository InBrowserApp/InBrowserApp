DNS Lookup ตรวจสอบ DNS record สาธารณะที่ส่งกลับมาสำหรับชื่อโดเมน เครื่องมือนี้มีประโยชน์เมื่อคุณกำลังตรวจสอบการเปิดตัวเว็บไซต์ใหม่ แก้ปัญหาการส่งอีเมล ตรวจสอบการเปลี่ยนแปลง CDN หรือโหลดบาลานเซอร์ หรือยืนยันว่าการตอบกลับที่เกี่ยวข้องกับ DNSSEC แตกต่างกันระหว่างรีโซลเวอร์หรือไม่

## ควรใช้เมื่อใด

ใช้เครื่องมือนี้เมื่อคุณต้องการคำตอบอย่างรวดเร็วจากฝั่งเบราว์เซอร์สำหรับประเภท DNS record ทั่วไป ระเบียน A และ AAAA แสดงปลายทาง IPv4 และ IPv6, ระเบียน CNAME แสดง alias, ระเบียน MX ระบุตัวแลกเปลี่ยนอีเมล, ระเบียน TXT มักมี SPF หรือโทเคนยืนยัน และระเบียน NS/SOA/CAA/SRV/HTTPS/SVCB เปิดเผยข้อมูลเกี่ยวกับ delegation, authority, certificate, service และคำใบ้ endpoint สมัยใหม่

## วิธีทำงาน

การค้นหาทำงานในเบราว์เซอร์ของคุณด้วย DNS over HTTPS เลือกรีโซลเวอร์ เลือกประเภทระเบียนอย่างน้อยหนึ่งประเภท แล้วส่งโดเมนหรือ URL ระบบจะแปลง URL ให้เหลือ hostname ก่อนส่งคำค้นหา ดังนั้นเมื่อวาง `https://www.example.com/path` ระบบจะค้นหา `www.example.com`

## การอ่านผลลัพธ์

แต่ละประเภทระเบียนจะแสดงแยกกันพร้อมรหัสตอบกลับ DNS, แฟล็กของรีโซลเวอร์, แถวคำตอบ และ JSON ดิบ `NoError` หมายความว่า DNS server ตอบกลับสำเร็จ แต่ยังอาจไม่มีแถวคำตอบสำหรับประเภทใดประเภทหนึ่งได้ `NXDomain`, `ServFail` หรือ `Refused` มักหมายความว่าชื่อนั้นไม่มีอยู่ รีโซลเวอร์ไม่สามารถค้นหาให้เสร็จได้ หรือนโยบายของรีโซลเวอร์บล็อกคำขอ

## ความเป็นส่วนตัวและข้อจำกัด

คำค้นหาจะถูกส่งไปยังรีโซลเวอร์ DNS over HTTPS ที่เลือก ไม่ได้ส่งไปยังเซิร์ฟเวอร์ของ InBrowser.App พฤติกรรมของรีโซลเวอร์ สถานะแคช การตรวจสอบ DNSSEC และการกรองของเครือข่ายในพื้นที่ล้วนส่งผลต่อผลลัพธ์ได้ เครื่องมือนี้ไม่ได้แทนที่การตรวจสอบ `dig` จาก authoritative server ในหลายเครือข่าย แต่เป็นวิธีที่รวดเร็วในการตรวจดูว่ารีโซลเวอร์ DoH สาธารณะส่งคืนอะไรจากเบราว์เซอร์ปัจจุบันของคุณ
