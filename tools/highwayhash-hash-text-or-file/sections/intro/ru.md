## Что такое HighwayHash?

HighwayHash — быстрая хеш-функция с ключом, разработанная Google для высокопроизводительного создания отпечатков и проверки целостности. Она использует 256-битный ключ и может создавать 64-битный, 128-битный или 256-битный вывод из одного и того же текста или файла.

## Когда его использовать

- Создавайте детерминированные контрольные суммы с ключом для ключей кэша, идентификаторов объектов, шардинга или внутренних таблиц поиска.
- Сравнивайте файлы или текстовые нагрузки с одним и тем же ключом, когда скорость важнее широкой криптографической совместимости.
- Создавайте 128-битные или 256-битные отпечатки, когда для сценариев проверки целостности нужен более крупный непарольный хеш.

## Параметры ключа и вывода

Введите ключ как ровно 32 байта шестнадцатеричных данных, например `0x000102030405060708090a0b0c0d0e0f101112131415161718191a1b1c1d1e1f`. Префикс `0x` необязателен, а инструмент принимает пробелы, двоеточия, дефисы и подчеркивания, чтобы длинные ключи было проще читать. Если оставить ключ пустым, будет использован ключ библиотеки по умолчанию: это удобно для быстрых проверок, но его не следует считать секретным.

## Примечания по безопасности

HighwayHash не заменяет HMAC, цифровые подписи или хеширование паролей. Используйте его для быстрых отпечатков с ключом и сценариев с контрольными суммами, а не для подтверждения подлинности между системами, которым нужна стандартная криптографическая проверка.
