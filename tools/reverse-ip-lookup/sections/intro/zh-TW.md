Reverse IP Lookup 會將 IPv4 或 IPv6 位址轉換成其反向 DNS 名稱，並查詢對應的 `PTR` 記錄。它可協助你檢查位址擁有者為郵件伺服器、網路設備、雲端執行個體和疑難排解紀錄發布的主機名稱。

## 檢查內容

對於 IPv4，工具會反轉各個八位元組的順序並查詢 `in-addr.arpa` 名稱。對於 IPv6，工具會將位址展開為 32 個十六進位半位元組，反轉其順序，並查詢相符的 `ip6.arpa` 名稱。結果會顯示確切的反向 DNS 網域、DNS 狀態碼、解析器、位址類型，以及所有傳回的主機名稱和其 TTL 值。

## 查詢如何執行

查詢會從你的瀏覽器使用 DNS-over-HTTPS 執行。你可以選擇 Cloudflare、Google 或 AliDNS 作為解析器，瀏覽器會向該端點傳送標準 `PTR` 查詢。不會涉及任何伺服器端的 InBrowser.App 查詢服務。

## 如何解讀缺少的結果

缺少 PTR 答案很常見。許多住宅、雲端、私有或新分配的位址不會發布反向 DNS 記錄。成功的 DNS 回應但沒有主機名稱，並不證明該位址未被使用；它只表示反向區域沒有透過所選解析器傳回可用的 `PTR` 記錄。

## 實務注意事項

- 反向 DNS 會將 IP 位址對應到主機名稱；這與找出託管在同一位址上的所有網域不同。
- PTR 記錄由 IP 位址擁有者或上游供應商控制，不只由網域擁有者控制。
- 郵件和安全系統通常會比對正向與反向 DNS，因此 PTR 記錄通常應指向會解析回同一位址的主機名稱。
