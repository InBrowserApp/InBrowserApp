역방향 IP 조회는 IPv4 또는 IPv6 주소를 역방향 DNS 이름으로 변환하고 해당 `PTR` 레코드를 쿼리합니다. 메일 서버, 네트워크 장비, 클라우드 인스턴스, 문제 해결 기록에서 주소 소유자가 어떤 호스트 이름을 게시했는지 확인하는 데 도움이 됩니다.

## 확인하는 항목

IPv4의 경우 이 도구는 옥텟 순서를 뒤집어 `in-addr.arpa` 이름을 쿼리합니다. IPv6의 경우 주소를 32개의 16진수 니블로 확장하고 순서를 뒤집어 일치하는 `ip6.arpa` 이름을 쿼리합니다. 결과에는 정확한 역방향 DNS 도메인, DNS 상태 코드, 리졸버, 주소 체계, 반환된 모든 호스트 이름과 TTL 값이 표시됩니다.

## 쿼리 실행 방식

조회는 DNS-over-HTTPS를 사용해 브라우저에서 실행됩니다. 리졸버로 Cloudflare, Google 또는 AliDNS를 선택할 수 있으며, 브라우저는 해당 엔드포인트로 표준 `PTR` 쿼리를 보냅니다. 서버 측 InBrowser.App 조회 서비스는 관여하지 않습니다.

## 누락된 결과를 해석하는 방법

PTR 응답이 없는 것은 흔한 일입니다. 많은 가정용, 클라우드, 사설 또는 새로 할당된 주소는 역방향 DNS 레코드를 게시하지 않습니다. 호스트 이름 없이 성공한 DNS 응답이 주소가 사용되지 않는다는 증거는 아닙니다. 선택한 리졸버를 통해 역방향 영역이 사용 가능한 `PTR` 레코드를 반환하지 않았다는 뜻일 뿐입니다.

## 실무 참고 사항

- 역방향 DNS는 IP 주소를 호스트 이름에 매핑합니다. 같은 주소에서 호스팅되는 모든 도메인을 찾는 것과는 다릅니다.
- PTR 레코드는 도메인 소유자만이 아니라 IP 주소 소유자 또는 상위 공급자가 제어합니다.
- 메일 및 보안 시스템은 정방향 DNS와 역방향 DNS를 비교하는 경우가 많으므로, PTR 레코드는 일반적으로 같은 주소로 다시 확인되는 호스트 이름을 가리켜야 합니다.
