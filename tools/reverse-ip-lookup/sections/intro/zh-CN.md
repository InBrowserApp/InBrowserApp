反向 IP 查询会将 IPv4 或 IPv6 地址转换为其反向 DNS 名称，并查询对应的 `PTR` 记录。它可以帮助你检查地址所有者为邮件服务器、网络设备、云实例和故障排查记录发布了哪个主机名。

## 检查内容

对于 IPv4，此工具会反转各个八位组，并查询 `in-addr.arpa` 名称。对于 IPv6，它会将地址展开为 32 个十六进制半字节，将其反转，然后查询匹配的 `ip6.arpa` 名称。结果会显示准确的反向 DNS 域名、DNS 状态码、解析器、地址族，以及返回的所有主机名及其 TTL 值。

## 查询如何运行

查询会在你的浏览器中使用 DNS-over-HTTPS 运行。你可以选择 Cloudflare、Google 或 AliDNS 作为解析器，浏览器会向该端点发送标准 `PTR` 查询。不会涉及任何服务器端 InBrowser.App 查询服务。

## 如何解读缺失结果

没有 PTR 应答很常见。许多住宅、云、私有或新分配的地址都不会发布反向 DNS 记录。DNS 响应成功但没有主机名，并不能证明该地址未被使用；它只表示反向区域未通过所选解析器返回可用的 `PTR` 记录。

## 实用说明

- 反向 DNS 将 IP 地址映射到主机名；这不同于查找托管在同一地址上的所有域名。
- PTR 记录由 IP 地址所有者或上游提供商控制，而不只由域名所有者控制。
- 邮件和安全系统经常比较正向和反向 DNS，因此 PTR 记录通常应指向一个可解析回同一地址的主机名。
