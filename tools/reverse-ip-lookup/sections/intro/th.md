Reverse IP Lookup จะแปลงที่อยู่ IPv4 หรือ IPv6 เป็นชื่อ reverse DNS แล้วสอบถาม `PTR` record ที่เกี่ยวข้อง เครื่องมือนี้ช่วยตรวจสอบว่าเจ้าของที่อยู่เผยแพร่ชื่อโฮสต์ใดไว้สำหรับเซิร์ฟเวอร์เมล อุปกรณ์เครือข่าย อินสแตนซ์คลาวด์ และบันทึกเพื่อการแก้ปัญหา

## สิ่งที่ตรวจสอบ

สำหรับ IPv4 เครื่องมือจะกลับลำดับอ็อกเท็ตแล้วสอบถามชื่อ `in-addr.arpa` สำหรับ IPv6 เครื่องมือจะขยายที่อยู่เป็น nibble เลขฐานสิบหก 32 ตัว กลับลำดับ แล้วสอบถามชื่อ `ip6.arpa` ที่ตรงกัน ผลลัพธ์จะแสดงโดเมน reverse DNS ที่แน่นอน สถานะ DNS, resolver, ตระกูลที่อยู่ และชื่อโฮสต์ที่ส่งกลับมาพร้อมค่า TTL

## การสอบถามทำงานอย่างไร

การค้นหาจะทำงานจากเบราว์เซอร์ของคุณโดยใช้ DNS-over-HTTPS คุณสามารถเลือก Cloudflare, Google หรือ AliDNS เป็น resolver ได้ และเบราว์เซอร์จะส่งคำขอ `PTR` มาตรฐานไปยัง endpoint นั้น ไม่มีบริการค้นหาฝั่งเซิร์ฟเวอร์ของ InBrowser.App เข้ามาเกี่ยวข้อง

## วิธีอ่านผลลัพธ์ที่ไม่พบ

การไม่มีคำตอบ PTR เป็นเรื่องปกติ ที่อยู่สำหรับผู้ใช้อยู่อาศัย คลาวด์ ส่วนตัว หรือที่เพิ่งได้รับมอบหมายจำนวนมากไม่ได้เผยแพร่ระเบียน reverse DNS การตอบกลับ DNS ที่สำเร็จแต่ไม่มีชื่อโฮสต์ไม่ได้พิสูจน์ว่าที่อยู่นั้นไม่ได้ใช้งาน หมายความเพียงว่า reverse zone ไม่ได้ส่งคืน `PTR` record ที่ใช้งานได้ผ่าน resolver ที่เลือก

## หมายเหตุเชิงปฏิบัติ

- Reverse DNS จับคู่ที่อยู่ IP กับชื่อโฮสต์ ซึ่งต่างจากการค้นหาโดเมนทั้งหมดที่โฮสต์อยู่บนที่อยู่เดียวกัน
- PTR records ถูกควบคุมโดยเจ้าของที่อยู่ IP หรือผู้ให้บริการอัปสตรีม ไม่ใช่เจ้าของโดเมนเพียงอย่างเดียว
- ระบบเมลและระบบความปลอดภัยมักเปรียบเทียบ forward DNS กับ reverse DNS ดังนั้น PTR record โดยทั่วไปควรชี้ไปยังชื่อโฮสต์ที่ resolve กลับมายังที่อยู่เดียวกัน
