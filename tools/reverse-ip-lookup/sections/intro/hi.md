Reverse IP Lookup किसी IPv4 या IPv6 पते को उसके reverse DNS नाम में बदलता है और संबंधित `PTR` रिकॉर्ड को क्वेरी करता है। यह देखने में मदद करता है कि मेल सर्वर, नेटवर्क उपकरण, क्लाउड इंस्टेंस और समस्या-निवारण नोट्स के लिए पते का मालिक कौन सा होस्टनाम प्रकाशित करता है।

## यह क्या जाँचता है

IPv4 के लिए, टूल octets को उलटता है और `in-addr.arpa` नाम को क्वेरी करता है। IPv6 के लिए, यह पते को 32 hexadecimal nibbles तक फैलाता है, उन्हें उलटता है, और मेल खाने वाले `ip6.arpa` नाम को क्वेरी करता है। परिणाम सटीक reverse DNS डोमेन, DNS status code, रिज़ॉल्वर, address family और लौटाए गए किसी भी होस्टनाम को उनके TTL मानों के साथ दिखाता है।

## क्वेरी कैसे चलती है

लुकअप DNS-over-HTTPS का उपयोग करके आपके ब्राउज़र से चलता है। आप रिज़ॉल्वर के रूप में Cloudflare, Google या AliDNS चुन सकते हैं, और ब्राउज़र उस endpoint पर एक मानक `PTR` query भेजता है। इसमें कोई server-side InBrowser.App lookup service शामिल नहीं है।

## न मिले परिणामों को कैसे पढ़ें

PTR उत्तर न मिलना सामान्य है। कई आवासीय, क्लाउड, निजी या हाल ही में आवंटित पते reverse DNS records प्रकाशित नहीं करते। बिना होस्टनाम वाला सफल DNS response यह साबित नहीं करता कि पता इस्तेमाल में नहीं है; इसका मतलब केवल इतना है कि reverse zone ने चुने गए resolver के माध्यम से उपयोगी `PTR` रिकॉर्ड नहीं लौटाया।

## व्यावहारिक नोट्स

- Reverse DNS किसी IP address को होस्टनाम से मैप करता है; यह उसी address पर hosted हर domain खोजने से अलग है।
- PTR records IP address owner या upstream provider द्वारा नियंत्रित होते हैं, सिर्फ domain owner द्वारा नहीं।
- मेल और सुरक्षा प्रणालियाँ अक्सर forward और reverse DNS की तुलना करती हैं, इसलिए PTR record आमतौर पर ऐसे hostname की ओर संकेत करना चाहिए जो उसी address पर वापस resolve होता हो।
