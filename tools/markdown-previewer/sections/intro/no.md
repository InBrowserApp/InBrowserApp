# Hva er Markdown?

Markdown er et lett oppmerkingsspråk for å skrive formatert tekst i vanlige tekstfiler. Den holder kilden lesbar samtidig som den støtter rik formatering når den gjengis.

**Vanlig syntaks:**

- `#` for overskrifter, `*` eller `_` for utheving, `-` for lister.
- Backticks for innebygd kode og inngjerdede kodeblokker for utdrag.
- `[text](url)` for lenker og `![alt](url)` for bilder.

Markdown er mye brukt for README-filer, dokumentasjon, notater og blogginnlegg. De fleste plattformer gjengir Markdown til HTML; CommonMark og GitHub Flavored Markdown legger til tabeller, oppgavelister og andre utvidelser. Filer bruker vanligvis utvidelsen `.md` eller `.markdown`, og rentekstformatet fungerer godt med versjonskontroll.

## Hvorfor bruke en forhåndsvisning?

- Gjennomgå overskrifter, tabeller, oppgavelister, kodegjerder og lenker før publisering.
- Fang innlimt HTML eller formateringsquirks uten å forlate nettleseren.
- Eksporter en frittstående HTML-fil når du trenger et utskriftsvennlig øyeblikksbilde.

## Hva dette verktøyet hjelper med

- README-filer, dokumenter, endringslogger, utgivelsesnotater og overleveringsinnhold.
- Notater, bloggutkast, interne veiledninger og kunnskapsbaserte artikler.
- Siste kontroller før du sender Markdown til et annet system eller arbeidsflyt for lagkamerater.

## Praktiske tips

- Hold sanitisering aktivert når Markdown inneholder HTML fra en ukjent kilde.
- Bruk konsistente overskriftsnivåer slik at den genererte oversikten forblir nyttig.
- Bytt til HTML-visningen når du trenger det eksakte eksporterte dokumentet.
