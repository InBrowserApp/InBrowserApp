# Cos'è Markdown?

Markdown è un linguaggio di markup leggero per scrivere testo formattato in file di testo semplice. Mantiene leggibile l'origine pur supportando la formattazione avanzata durante il rendering.

**Sintassi comune:**

- `#` per intestazioni, `*` o `_` per enfasi, `-` per elenchi.
- Backtick per codice in linea e blocchi di codice recintato per snippet.
- `[text](url)` per i collegamenti e `![alt](url)` per le immagini.

Markdown è ampiamente utilizzato per file README, documentazione, note e post di blog. La maggior parte delle piattaforme esegue il rendering di Markdown in HTML; CommonMark e GitHub Flavored Markdown aggiungono tabelle, elenchi di attività e altre estensioni. I file di solito utilizzano l'estensione `.md` o `.markdown` e il formato testo normale funziona bene con il controllo della versione.

## Perché utilizzare un visualizzatore di anteprima?

- Esamina intestazioni, tabelle, elenchi di attività, recinzioni di codice e collegamenti prima della pubblicazione.
- Rileva i codici HTML incollati o le peculiarità della formattazione senza uscire dal browser.
- Esporta un file HTML autonomo quando hai bisogno di un'istantanea facile da stampare.

## In cosa aiuta questo strumento

- File README, documenti, log delle modifiche, note di rilascio e contenuti di trasferimento.
- Note, bozze di blog, guide interne e articoli della knowledge base.
- Controlli finali prima di inviare Markdown a un altro sistema o flusso di lavoro di un compagno di squadra.

## Consigli pratici

- Mantieni abilitata la sanificazione quando Markdown contiene HTML proveniente da una fonte sconosciuta.
- Utilizza livelli di intestazione coerenti in modo che la struttura generata rimanga utile.
- Passa alla visualizzazione HTML quando hai bisogno dell'esatto documento esportato.
