# What is Markdown?

Markdown is a lightweight markup language for writing formatted text in plain text files. It keeps the source readable while still supporting rich formatting when rendered.

**Common syntax:**

- `#` for headings, `*` or `_` for emphasis, `-` for lists.
- Backticks for inline code and fenced code blocks for snippets.
- `[text](url)` for links and `![alt](url)` for images.

Markdown is widely used for README files, documentation, notes, and blog posts. Most platforms render Markdown to HTML; CommonMark and GitHub Flavored Markdown add tables, task lists, and other extensions. Files usually use the `.md` or `.markdown` extension, and the plain-text format works well with version control.

## Why Use A Previewer?

- Review headings, tables, task lists, code fences, and links before publishing.
- Catch pasted HTML or formatting quirks without leaving the browser.
- Export a standalone HTML file when you need a print-friendly snapshot.

## Practical Tips

- Keep sanitization enabled when the Markdown contains HTML from an unknown source.
- Use consistent heading levels so the generated outline stays useful.
- Switch to the HTML view when you need the exact exported document.
