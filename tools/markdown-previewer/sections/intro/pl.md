# Co to jest Markdown?

Markdown to lekki język znaczników do zapisywania sformatowanego tekstu w zwykłych plikach tekstowych. Zapewnia czytelność źródła, jednocześnie obsługując bogate formatowanie podczas renderowania.

**Wspólna składnia:**

- `#` dla nagłówków, `*` lub `_` dla wyróżnień, `-` dla list.
- Backticki dla kodu wbudowanego i chronione bloki kodu dla fragmentów.
- `[text](url)` dla linków i `![alt](url)` dla obrazów.

Markdown jest szeroko stosowany w plikach README, dokumentacji, notatkach i postach na blogu. Większość platform renderuje Markdown do formatu HTML; CommonMark i GitHub Flavoured Markdown dodają tabele, listy zadań i inne rozszerzenia. Pliki zwykle korzystają z rozszerzenia `.md` lub `.markdown`, a format zwykłego tekstu dobrze współpracuje z kontrolą wersji.

## Dlaczego warto korzystać z podglądu?

- Przed publikacją przejrzyj nagłówki, tabele, listy zadań, zabezpieczenia kodu i łącza.
- Wychwytuj wklejony kod HTML lub dziwactwa formatowania bez opuszczania przeglądarki.
- Eksportuj samodzielny plik HTML, jeśli potrzebujesz migawki do wydrukowania.

## W czym pomaga to narzędzie

- Pliki README, dokumenty, dzienniki zmian, informacje o wersji i treść przekazania.
- Notatki, wersje robocze blogów, wewnętrzne przewodniki i artykuły z bazy wiedzy.
- Końcowe kontrole przed wysłaniem Markdown do innego systemu lub przepływu pracy członka zespołu.

## Praktyczne wskazówki

- Włącz oczyszczanie, gdy Markdown zawiera kod HTML z nieznanego źródła.
- Używaj spójnych poziomów nagłówków, aby wygenerowany konspekt pozostał użyteczny.
- Jeśli potrzebujesz dokładnie wyeksportowanego dokumentu, przełącz się na widok HTML.
