# Что такое Маркдаун?

Markdown — это легкий язык разметки для записи форматированного текста в обычные текстовые файлы. Он сохраняет исходный код читабельным, сохраняя при этом поддержку расширенного форматирования при рендеринге.

**Общий синтаксис:**

- `#` для заголовков, `*` или `_` для выделения, `-` для списков.
- Обратные кавычки для встроенного кода и изолированные блоки кода для фрагментов.
- `[text](url)` для ссылок и `![alt](url)` для изображений.

Markdown широко используется для файлов README, документации, заметок и сообщений в блогах. Большинство платформ преобразуют Markdown в HTML; CommonMark и GitHub Flavored Markdown добавляют таблицы, списки задач и другие расширения. Файлы обычно используют расширение `.md` или `.markdown`, а текстовый формат хорошо работает с контролем версий.

## Зачем использовать программу предварительного просмотра?

- Перед публикацией просмотрите заголовки, таблицы, списки задач, границы кода и ссылки.
- Улавливайте вставленный HTML-код или особенности форматирования, не выходя из браузера.
- Экспортируйте отдельный HTML-файл, если вам нужен снимок, удобный для печати.

## В чем помогает этот инструмент

- Файлы README, документы, журналы изменений, примечания к выпуску и передаваемый контент.
- Заметки, черновики блогов, внутренние руководства и статьи базы знаний.
- Заключительные проверки перед отправкой Markdown в другую систему или рабочий процесс товарища по команде.

## Практические советы

- Оставляйте включенной очистку, если Markdown содержит HTML из неизвестного источника.
- Используйте одинаковые уровни заголовков, чтобы созданный план оставался полезным.
- Переключитесь в режим HTML, когда вам понадобится именно тот экспортированный документ.
