# Vad är Markdown?

Markdown är ett lättviktigt märkningsspråk för att skriva formaterad text i vanliga textfiler. Det håller källan läsbar samtidigt som den stöder rik formatering när den renderas.

**Vanlig syntax:**

- `#` för rubriker, `*` eller `_` för betoning, `-` för listor.
- Backticks för inline-kod och inhägnade kodblock för utdrag.
- `[text](url)` för länkar och `![alt](url)` för bilder.

Markdown används ofta för README-filer, dokumentation, anteckningar och blogginlägg. De flesta plattformar gör Markdown till HTML; CommonMark och GitHub Flavored Markdown lägger till tabeller, uppgiftslistor och andra tillägg. Filer använder vanligtvis tillägget `.md` eller `.markdown`, och vanlig textformat fungerar bra med versionskontroll.

## Varför använda en förhandsgranskare?

- Granska rubriker, tabeller, uppgiftslistor, kodstängsel och länkar innan du publicerar.
- Fånga inklistrade HTML- eller formateringsdetaljer utan att lämna webbläsaren.
- Exportera en fristående HTML-fil när du behöver en utskriftsvänlig ögonblicksbild.

## Vad det här verktyget hjälper med

- README-filer, dokument, ändringsloggar, release notes och handoff-innehåll.
- Anteckningar, bloggutkast, interna guider och kunskapsbaserade artiklar.
- Slutkontroller innan du skickar Markdown till ett annat system eller arbetsflöde för teamkamrater.

## Praktiska tips

- Håll sanering aktiverad när Markdown innehåller HTML från en okänd källa.
- Använd konsekventa rubriknivåer så att den genererade dispositionen förblir användbar.
- Växla till HTML-vyn när du behöver det exakta exporterade dokumentet.
