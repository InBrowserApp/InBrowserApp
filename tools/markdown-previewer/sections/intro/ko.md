# 마크다운이란 무엇입니까?

Markdown은 일반 텍스트 파일에 서식 있는 텍스트를 작성하기 위한 경량 마크업 언어입니다. 렌더링 시 다양한 형식을 지원하면서 소스를 읽을 수 있게 유지합니다.

**공통 구문:**

- 제목의 경우 `#`, 강조의 경우 `*` 또는 `_`, 목록의 경우 `-`입니다.
- 인라인 코드의 백틱과 스니펫의 분리 코드 블록입니다.
- 링크의 경우 `[text](url)`, 이미지의 경우 `![alt](url)`입니다.

Markdown은 README 파일, 문서, 메모 및 블로그 게시물에 널리 사용됩니다. 대부분의 플랫폼은 Markdown을 HTML로 렌더링합니다. CommonMark 및 GitHub Flavored Markdown은 테이블, 작업 목록 및 기타 확장을 추가합니다. 파일은 일반적으로 `.md` 또는 `.markdown` 확장자를 사용하며 일반 텍스트 형식은 버전 제어와 잘 작동합니다.

## 미리보기를 사용하는 이유는 무엇입니까?

- 게시하기 전에 제목, 표, 작업 목록, 코드 펜스 및 링크를 검토하세요.
- 브라우저를 떠나지 않고도 붙여넣은 HTML이나 서식 관련 문제를 포착할 수 있습니다.
- 인쇄용 스냅샷이 필요한 경우 독립형 HTML 파일을 내보냅니다.

## 이 도구의 도움

- README 파일, 문서, 변경 로그, 릴리스 노트 및 핸드오프 콘텐츠.
- 메모, 블로그 초안, 내부 가이드 및 지식 기반 기사.
- Markdown을 다른 시스템이나 팀원 워크플로로 보내기 전에 최종 확인합니다.

## 실용적인 팁

- 마크다운에 알 수 없는 소스의 HTML이 포함된 경우 삭제를 활성화된 상태로 유지하세요.
- 생성된 개요가 유용하게 유지되도록 일관된 제목 수준을 사용합니다.
- 정확한 내보낸 문서가 필요한 경우 HTML 보기로 전환하세요.
