# ما هو Markdown؟

Markdown هي لغة ترميزية خفيفة الوزن لكتابة نص منسق في ملفات نصية عادية. فهو يبقي المصدر قابلاً للقراءة مع الاستمرار في دعم التنسيق الغني عند عرضه.

**البنية الشائعة:**

- `#` للعناوين، `*` أو `_` للتأكيد، `-` للقوائم.
- تُستخدم العلامات المائلة العكسية للكود المضمن، وتُستخدم كتل الكود المسوّرة للمقتطفات.
- `[text](url)` للروابط و`![alt](url)` للصور.

يتم استخدام Markdown على نطاق واسع لملفات README والوثائق والملاحظات ومنشورات المدونة. تعرض معظم الأنظمة الأساسية Markdown إلى HTML؛ يضيف CommonMark وGitHub Flavoured Markdown الجداول وقوائم المهام والإضافات الأخرى. تستخدم الملفات عادةً الامتداد `.md` أو `.markdown`، ويعمل تنسيق النص العادي بشكل جيد مع التحكم في الإصدار.

## لماذا تستخدم المعاينة؟

- قم بمراجعة العناوين والجداول وقوائم المهام وأسوار التعليمات البرمجية والروابط قبل النشر.
- التقط مشاكل HTML الملصق أو غرابة التنسيق من دون مغادرة المتصفح.
- قم بتصدير ملف HTML مستقل عندما تحتاج إلى لقطة سهلة الطباعة.

## فيمَ تساعد هذه الأداة؟

- ملفات README والمستندات وسجلات التغيير وملاحظات الإصدار ومحتوى التسليم.
- الملاحظات ومسودات المدونة والأدلة الداخلية ومقالات قاعدة المعرفة.
- الفحوصات النهائية قبل إرسال Markdown إلى نظام آخر أو سير عمل زميل في الفريق.

## نصائح عملية

- حافظ على تمكين التعقيم عندما يحتوي Markdown على HTML من مصدر غير معروف.
- استخدم مستويات عناوين متسقة حتى يظل المخطط التفصيلي الذي تم إنشاؤه مفيدًا.
- قم بالتبديل إلى عرض HTML عندما تحتاج إلى المستند الذي تم تصديره بالضبط.
