# O que é Markdown?

Markdown é uma linguagem de marcação leve para escrever texto formatado em arquivos de texto simples. Ele mantém a fonte legível e ainda oferece suporte à formatação avançada quando renderizada.

**Sintaxe comum:**

- `#` para títulos, `*` ou `_` para ênfase, `-` para listas.
- Backticks para código embutido e blocos de código protegidos para snippets.
- `[text](url)` para links e `![alt](url)` para imagens.

Markdown é amplamente usado para arquivos README, documentação, notas e postagens de blog. A maioria das plataformas renderiza Markdown em HTML; CommonMark e GitHub Flavored Markdown adicionam tabelas, listas de tarefas e outras extensões. Os arquivos geralmente usam a extensão `.md` ou `.markdown`, e o formato de texto simples funciona bem com controle de versão.

## Por que usar um visualizador?

- Revise títulos, tabelas, listas de tarefas, limites de código e links antes de publicar.
- Capture HTML colado ou peculiaridades de formatação sem sair do navegador.
- Exporte um arquivo HTML independente quando precisar de um instantâneo para impressão.

## Em que esta ferramenta ajuda

- Arquivos README, documentos, changelogs, notas de versão e conteúdo de transferência.
- Notas, rascunhos de blog, guias internos e artigos da base de conhecimento.
- Verificações finais antes de enviar Markdown para outro sistema ou fluxo de trabalho de colega de equipe.

## Dicas Práticas

- Mantenha a sanitização habilitada quando o Markdown contiver HTML de uma fonte desconhecida.
- Use níveis de título consistentes para que o contorno gerado permaneça útil.
- Mude para a visualização HTML quando precisar do documento exportado exato.
