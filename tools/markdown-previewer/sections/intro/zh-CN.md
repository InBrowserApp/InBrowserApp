# 什么是 Markdown？

Markdown 是一种轻量级标记语言，用于在纯文本文件中编写带格式的内容。它既保留了源文本的可读性，也能在渲染后呈现更丰富的排版效果。

**常用语法：**

- `#` 表示标题，`*` 或 `_` 表示强调，`-` 表示列表。
- 反引号用于行内代码，围栏代码块用于展示代码片段。
- `[text](url)` 用于链接，`![alt](url)` 用于图片。

Markdown 广泛用于 README、文档、笔记和博客文章。大多数平台会将 Markdown 渲染为 HTML；CommonMark 和 GitHub Flavored Markdown 还支持表格、任务列表等扩展。文件通常使用 `.md` 或 `.markdown` 扩展名，纯文本格式也很适合纳入版本控制。

## 为什么使用预览器？

- 在发布前快速检查标题、表格、任务列表、代码块和链接是否渲染正确。
- 不离开浏览器就能发现复制粘贴带来的 HTML 残留或格式问题。
- 需要打印或分享静态结果时，可以直接导出独立 HTML 文件。

## 这个工具有什么帮助

- README、文档、更新日志、发布说明和交接材料。
- 笔记、博客草稿、内部指南和知识库文章。
- 在把 Markdown 发给其他系统或协作者之前做最后检查。

## 实用技巧

- 如果 Markdown 中混入了来源不明的 HTML，建议保持“清理 HTML”开启。
- 尽量使用连续且一致的标题层级，这样生成的目录会更清晰。
- 当你需要查看最终导出的结构时，可以切换到 HTML 视图。
