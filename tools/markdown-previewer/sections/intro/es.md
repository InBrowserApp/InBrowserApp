# ¿Qué es Markdown?

Markdown es un lenguaje de marcado ligero para escribir texto formateado en archivos de texto sin formato. Mantiene la fuente legible y al mismo tiempo admite formato enriquecido cuando se procesa.

**Sintaxis común:**

- `#` para encabezados, `*` o `_` para énfasis, `-` para listas.
- Comillas invertidas para código en línea y bloques de código delimitados para fragmentos.
- `[text](url)` para enlaces y `![alt](url)` para imágenes.

Markdown se usa ampliamente para archivos README, documentación, notas y publicaciones de blogs. La mayoría de las plataformas convierten Markdown en HTML; CommonMark y GitHub Flavored Markdown agregan tablas, listas de tareas y otras extensiones. Los archivos suelen utilizar la extensión `.md` o `.markdown` y el formato de texto sin formato funciona bien con el control de versiones.

## ¿Por qué utilizar una vista previa?

- Revise encabezados, tablas, listas de tareas, bloques de código y enlaces antes de publicar.
- Capture el HTML pegado o las peculiaridades del formato sin salir del navegador.
- Exporte un archivo HTML independiente cuando necesite una instantánea para imprimir.

## Con qué ayuda esta herramienta

- Archivos README, documentos, registros de cambios, notas de la versión y contenido de transferencia.
- Notas, borradores de blogs, guías internas y artículos de base de conocimientos.
- Comprobaciones finales antes de enviar Markdown a otro sistema o flujo de trabajo de un compañero de equipo.

## Consejos prácticos

- Mantenga habilitada la desinfección cuando Markdown contenga HTML de una fuente desconocida.
- Utilice niveles de encabezado consistentes para que el esquema generado siga siendo útil.
- Cambie a la vista HTML cuando necesite el documento exportado exacto.
