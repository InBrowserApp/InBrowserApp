# Was ist Markdown?

Markdown ist eine leichte Auszeichnungssprache zum Schreiben von formatiertem Text in reine Textdateien. Dadurch bleibt die Quelle lesbar, während beim Rendern weiterhin umfangreiche Formatierung unterstützt wird.

**Gemeinsame Syntax:**

- `#` für Überschriften, `*` oder `_` für Hervorhebungen, `-` für Listen.
- Backticks für Inline-Code und eingezäunte Codeblöcke für Snippets.
- `[text](url)` für Links und `![alt](url)` für Bilder.

Markdown wird häufig für README-Dateien, Dokumentationen, Notizen und Blogbeiträge verwendet. Die meisten Plattformen rendern Markdown in HTML; CommonMark und GitHub Flavored Markdown fügen Tabellen, Aufgabenlisten und andere Erweiterungen hinzu. Dateien verwenden normalerweise die Erweiterung `.md` oder `.markdown`, und das Nur-Text-Format funktioniert gut mit der Versionskontrolle.

## Warum einen Previewer verwenden?

- Überprüfen Sie vor der Veröffentlichung Überschriften, Tabellen, Aufgabenlisten, Code-Zäune und Links.
- Erkennen Sie eingefügtes HTML oder Formatierungsfehler, ohne den Browser zu verlassen.
- Exportieren Sie eine eigenständige HTML-Datei, wenn Sie einen druckfreundlichen Schnappschuss benötigen.

## Wobei dieses Tool hilft

- README-Dateien, Dokumente, Änderungsprotokolle, Versionshinweise und Übergabeinhalte.
- Notizen, Blog-Entwürfe, interne Leitfäden und Wissensdatenbankartikel.
- Letzte Prüfungen, bevor Markdown an ein anderes System oder einen Teamkollegen-Workflow gesendet wird.

## Praktische Tipps

- Lassen Sie die Bereinigung aktiviert, wenn der Markdown HTML aus einer unbekannten Quelle enthält.
- Verwenden Sie konsistente Überschriftenebenen, damit die generierte Gliederung nützlich bleibt.
- Wechseln Sie zur HTML-Ansicht, wenn Sie das exakte exportierte Dokument benötigen.
