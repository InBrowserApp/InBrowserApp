# 什麼是 Markdown？

Markdown 是一種輕量級標記語言，用於在純文字檔案中撰寫帶格式的內容。它保留原始文字的可讀性，同時也能在渲染後呈現更豐富的排版效果。

**常用語法：**

- `#` 表示標題，`*` 或 `_` 表示強調，`-` 表示清單。
- 反引號用於行內程式碼，圍欄程式碼區塊用於展示程式片段。
- `[text](url)` 用於連結，`![alt](url)` 用於圖片。

Markdown 廣泛用於 README、文件、筆記與部落格文章。多數平台會將 Markdown 渲染為 HTML；CommonMark 與 GitHub Flavored Markdown 也支援表格、任務清單等擴充。檔案通常使用 `.md` 或 `.markdown` 副檔名，純文字格式也很適合納入版本控制。

## 為什麼要使用預覽器？

- 在發布前快速檢查標題、表格、任務清單、程式碼區塊與連結是否渲染正確。
- 不離開瀏覽器也能發現貼上內容帶來的 HTML 殘留或格式問題。
- 需要列印或分享靜態結果時，可以直接匯出獨立的 HTML 檔案。

## 這個工具有什麼幫助

- README、文件、變更日誌、版本說明與交接內容。
- 筆記、部落格草稿、內部指南與知識庫文章。
- 在把 Markdown 送進其他系統或交給協作者之前做最後檢查。

## 實用技巧

- 如果 Markdown 內含來自未知來源的 HTML，建議保持「清理 HTML」開啟。
- 盡量使用連續且一致的標題層級，讓產生出的目錄更清楚。
- 當你需要查看最終匯出的結構時，可以切換到 HTML 檢視。
