# מה זה Markdown?

Markdown היא שפת סימון קלת משקל לכתיבת טקסט מעוצב בקובצי טקסט רגיל. זה שומר על המקור קריא ועדיין תומך בעיצוב עשיר בעת רינדור.

**תחביר נפוץ:**

- `#` לכותרות, `*` או `_` להדגשה, `-` לרשימות.
- התקפות אחוריות עבור קוד מוטבע ובלוקי קוד מגודרים עבור קטעים.
- `[text](url)` לקישורים ו-`![alt](url)` לתמונות.

Markdown נמצא בשימוש נרחב עבור קבצי README, תיעוד, הערות ופוסטים בבלוג. רוב הפלטפורמות מעבדות את Markdown ל-HTML; CommonMark ו-GitHub Flavored Markdown מוסיפים טבלאות, רשימות משימות והרחבות אחרות. קבצים משתמשים בדרך כלל בסיומת `.md` או `.markdown`, ופורמט הטקסט הפשוט עובד היטב עם בקרת גרסאות.

## למה להשתמש בתצוגה מקדימה?

- סקור כותרות, טבלאות, רשימות משימות, גדרות קוד וקישורים לפני הפרסום.
- תפוס HTML מודבק או מוזרויות עיצוב מבלי לצאת מהדפדפן.
- ייצא קובץ HTML עצמאי כאשר אתה צריך תמונת מצב ידידותית להדפסה.

## במה הכלי הזה עוזר

- קבצי README, מסמכים, יומני שינויים, הערות שחרור ותוכן מסירה.
- הערות, טיוטות בלוג, מדריכים פנימיים ומאמרים מבוססי ידע.
- בדיקות אחרונות לפני שליחת Markdown למערכת אחרת או לזרימת עבודה של חבר לצוות.

## טיפים מעשיים

- שמור על חיטוי מופעל כאשר ה-Markdown מכיל HTML ממקור לא ידוע.
- השתמש ברמות כותרת עקביות כדי שהמתאר שנוצר יישאר שימושי.
- עבור לתצוגת HTML כאשר אתה צריך את המסמך המיוצא בדיוק.
