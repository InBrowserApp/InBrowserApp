# Qu’est-ce que le Markdown ?

Markdown est un langage de balisage léger permettant d'écrire du texte formaté dans des fichiers de texte brut. Il maintient la source lisible tout en prenant en charge un formatage riche lors du rendu.

**Syntaxe commune :**

- `#` pour les titres, `*` ou `_` pour l'emphase, `-` pour les listes.
- Backticks pour le code en ligne et blocs de code clôturés pour les extraits de code.
- `[text](url)` pour les liens et `![alt](url)` pour les images.

Markdown est largement utilisé pour les fichiers README, la documentation, les notes et les articles de blog. La plupart des plates-formes rendent Markdown au format HTML ; CommonMark et GitHub Flavored Markdown ajoutent des tableaux, des listes de tâches et d'autres extensions. Les fichiers utilisent généralement l'extension `.md` ou `.markdown`, et le format de texte brut fonctionne bien avec le contrôle de version.

## Pourquoi utiliser un outil de prévisualisation ?

- Examinez les titres, les tableaux, les listes de tâches, les barrières de code et les liens avant de publier.
- Capturez les bizarreries HTML ou de formatage collées sans quitter le navigateur.
- Exportez un fichier HTML autonome lorsque vous avez besoin d’un instantané imprimable.

## À quoi sert cet outil

- Fichiers README, documents, journaux des modifications, notes de version et contenu de transfert.
- Notes, brouillons de blog, guides internes et articles de la base de connaissances.
- Vérifications finales avant d'envoyer Markdown dans un autre système ou flux de travail d'un coéquipier.

## Conseils pratiques

- Gardez la désinfection activée lorsque le Markdown contient du HTML provenant d'une source inconnue.
- Utilisez des niveaux de titre cohérents pour que le plan généré reste utile.
- Basculez vers la vue HTML lorsque vous avez besoin du document exporté exact.
