# มาร์กดาวน์คืออะไร?

Markdown เป็นภาษามาร์กอัปน้ำหนักเบาสำหรับการเขียนข้อความที่จัดรูปแบบในไฟล์ข้อความธรรมดา ช่วยให้ต้นฉบับสามารถอ่านได้ในขณะที่ยังคงสนับสนุนการจัดรูปแบบที่หลากหลายเมื่อแสดงผล

**ไวยากรณ์ทั่วไป:**

- `#` สำหรับส่วนหัว, `*` หรือ `_` สำหรับการเน้น, `-` สำหรับรายการ
- Backticks สำหรับโค้ดแบบอินไลน์และบล็อกโค้ดแบบรั้วสำหรับตัวอย่างข้อมูล
- `[text](url)` สำหรับลิงก์และ `![alt](url)` สำหรับรูปภาพ

Markdown ใช้กันอย่างแพร่หลายสำหรับไฟล์ README เอกสารประกอบ บันทึกย่อ และโพสต์ในบล็อก แพลตฟอร์มส่วนใหญ่เรนเดอร์ Markdown เป็น HTML; CommonMark และ GitHub Flavoured Markdown เพิ่มตาราง รายการงาน และส่วนขยายอื่นๆ โดยปกติไฟล์จะใช้ส่วนขยาย `.md` หรือ `.markdown` และรูปแบบข้อความธรรมดาจะทำงานได้ดีกับการควบคุมเวอร์ชัน

## เหตุใดจึงต้องใช้โปรแกรมดูตัวอย่าง?

- ตรวจสอบส่วนหัว ตาราง รายการงาน รั้วโค้ด และลิงก์ก่อนเผยแพร่
- จับ HTML ที่วางหรือการจัดรูปแบบที่แปลกประหลาดโดยไม่ต้องออกจากเบราว์เซอร์
- ส่งออกไฟล์ HTML แบบสแตนด์อโลนเมื่อคุณต้องการสแน็ปช็อตที่เหมาะกับการพิมพ์

## เครื่องมือนี้ช่วยอะไรได้บ้าง

- ไฟล์ README เอกสาร บันทึกการเปลี่ยนแปลง บันทึกประจำรุ่น และเนื้อหาแฮนด์ออฟ
- หมายเหตุ ร่างบล็อก คำแนะนำภายใน และบทความฐานความรู้
- การตรวจสอบขั้นสุดท้ายก่อนส่ง Markdown ไปยังระบบอื่นหรือเวิร์กโฟลว์เพื่อนร่วมทีม

## เคล็ดลับการปฏิบัติ

- เปิดใช้งานการฆ่าเชื้อต่อไปเมื่อ Markdown มี HTML จากแหล่งที่ไม่รู้จัก
- ใช้ระดับหัวเรื่องที่สอดคล้องกันเพื่อให้โครงร่างที่สร้างขึ้นยังคงมีประโยชน์
- สลับไปที่มุมมอง HTML เมื่อคุณต้องการเอกสารที่ส่งออกทุกประการ
