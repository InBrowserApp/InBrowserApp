# Apakah Markdown?

Markdown ialah bahasa penanda yang ringan untuk menulis teks berformat dalam fail teks biasa. Ia memastikan sumber boleh dibaca sambil masih menyokong pemformatan kaya apabila diberikan.

**Sintaks biasa:**

- `#` untuk tajuk, `*` atau `_` untuk penekanan, `-` untuk senarai.
- Tanda belakang untuk kod sebaris dan blok kod berpagar untuk coretan.
- `[text](url)` untuk pautan dan `![alt](url)` untuk imej.

Markdown digunakan secara meluas untuk fail README, dokumentasi, nota dan catatan blog. Kebanyakan platform menjadikan Markdown kepada HTML; CommonMark dan GitHub Flavoured Markdown menambah jadual, senarai tugas dan sambungan lain. Fail biasanya menggunakan sambungan `.md` atau `.markdown` dan format teks biasa berfungsi dengan baik dengan kawalan versi.

## Mengapa Menggunakan Pratonton?

- Semak tajuk, jadual, senarai tugas, pagar kod dan pautan sebelum diterbitkan.
- Tangkap HTML yang ditampal atau kebiasaan pemformatan tanpa meninggalkan penyemak imbas.
- Eksport fail HTML kendiri apabila anda memerlukan syot kilat mesra cetak.

## Perkara yang Dibantu oleh Alat Ini

- Fail README, dokumen, log perubahan, nota keluaran dan kandungan penyerahan.
- Nota, draf blog, panduan dalaman dan artikel asas pengetahuan.
- Pemeriksaan akhir sebelum menghantar Markdown ke sistem lain atau aliran kerja rakan sepasukan.

## Petua Praktikal

- Pastikan sanitasi didayakan apabila Markdown mengandungi HTML daripada sumber yang tidak diketahui.
- Gunakan tahap tajuk yang konsisten supaya garis besar yang dijana kekal berguna.
- Beralih kepada paparan HTML apabila anda memerlukan dokumen yang dieksport dengan tepat.
