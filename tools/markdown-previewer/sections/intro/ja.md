# マークダウンとは何ですか?

Markdown は、書式設定されたテキストをプレーン テキスト ファイルに書き込むための軽量のマークアップ言語です。ソースを読みやすい状態に保ちながら、レンダリング時にリッチな書式設定をサポートします。

**一般的な構文:**

- 見出しの場合は `#`、強調の場合は `*` または `_`、リストの場合は `-`。
- インライン コードのバッククォートとスニペットのフェンスで囲まれたコード ブロック。
- リンクの場合は `[text](url)`、画像の場合は `![alt](url)`。

Markdown は、README ファイル、ドキュメント、メモ、ブログ投稿に広く使用されています。ほとんどのプラットフォームは Markdown を HTML にレンダリングします。 CommonMark と GitHub Flavored Markdown は、テーブル、タスク リスト、その他の拡張機能を追加します。ファイルは通常、`.md` または `.markdown` 拡張子を使用し、プレーンテキスト形式はバージョン管理で適切に機能します。

## プレビューアを使用する理由

- 公開する前に、見出し、表、タスク リスト、コード フェンス、リンクを確認してください。
- ブラウザを離れることなく、貼り付けられた HTML や書式設定の癖を見つけます。
- 印刷用のスナップショットが必要な場合は、スタンドアロン HTML ファイルをエクスポートします。

## このツールが役立つこと

- README ファイル、ドキュメント、変更ログ、リリース ノート、および引き継ぎコンテンツ。
- メモ、ブログの下書き、内部ガイド、ナレッジベースの記事。
- Markdown を別のシステムまたはチームメイトのワークフローに送信する前の最終チェック。

## 実践的なヒント

- マークダウンに不明なソースからの HTML が含まれている場合は、サニタイズを有効にしてください。
- 生成されたアウトラインが有用であり続けるように、一貫した見出しレベルを使用します。
- 正確にエクスポートされたドキュメントが必要な場合は、HTML ビューに切り替えます。
