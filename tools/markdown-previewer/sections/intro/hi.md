# मार्कडाउन क्या है?

सादे पाठ फ़ाइलों में स्वरूपित पाठ लिखने के लिए मार्कडाउन एक हल्की मार्कअप भाषा है। यह रेंडर किए जाने पर रिच फ़ॉर्मेटिंग का समर्थन करते हुए भी स्रोत को पठनीय रखता है।

**सामान्य वाक्यविन्यास:**

- शीर्षकों के लिए `#`, जोर देने के लिए `*` या `_`, सूचियों के लिए `-`।
- इनलाइन कोड के लिए बैकटिक्स और स्निपेट्स के लिए फेंसिड कोड ब्लॉक।
- लिंक के लिए `[text](url)` और छवियों के लिए `![alt](url)`।

मार्कडाउन का व्यापक रूप से README फ़ाइलों, दस्तावेज़ीकरण, नोट्स और ब्लॉग पोस्ट के लिए उपयोग किया जाता है। अधिकांश प्लेटफ़ॉर्म मार्कडाउन को HTML में प्रस्तुत करते हैं; कॉमनमार्क और गिटहब फ्लेवर्ड मार्कडाउन टेबल, कार्य सूचियां और अन्य एक्सटेंशन जोड़ते हैं। फ़ाइलें आमतौर पर `.md` या `.markdown` एक्सटेंशन का उपयोग करती हैं, और सादा-पाठ प्रारूप संस्करण नियंत्रण के साथ अच्छी तरह से काम करता है।

## पूर्वावलोकनकर्ता का उपयोग क्यों करें?

- प्रकाशन से पहले शीर्षकों, तालिकाओं, कार्य सूचियों, कोड बाड़ों और लिंक की समीक्षा करें।
- ब्राउज़र छोड़े बिना चिपकाए गए HTML या फ़ॉर्मेटिंग संबंधी विचित्रताओं को पकड़ें।
- जब आपको प्रिंट-अनुकूल स्नैपशॉट की आवश्यकता हो तो एक स्टैंडअलोन HTML फ़ाइल निर्यात करें।

## यह टूल किसमें मदद करता है

- README फ़ाइलें, दस्तावेज़, चेंजलॉग, रिलीज़ नोट्स और हैंडऑफ़ सामग्री।
- नोट्स, ब्लॉग ड्राफ्ट, आंतरिक गाइड और ज्ञान-आधारित लेख।
- मार्कडाउन को किसी अन्य सिस्टम या टीममेट वर्कफ़्लो में भेजने से पहले अंतिम जाँच।

## व्यावहारिक युक्तियाँ

- जब मार्कडाउन में किसी अज्ञात स्रोत से HTML शामिल हो तो सैनिटाइजेशन सक्षम रखें।
- सुसंगत शीर्षक स्तरों का उपयोग करें ताकि उत्पन्न रूपरेखा उपयोगी बनी रहे।
- जब आपको सटीक निर्यातित दस्तावेज़ की आवश्यकता हो तो HTML दृश्य पर स्विच करें।
