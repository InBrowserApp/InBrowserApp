## ما هي عملية التحويل بين JWK و PEM؟

JWK ‏(JSON Web Key) هو تنسيق JSON لمفاتيح التشفير يُستخدم في أنظمة JOSE/JWT. يمكنه تمثيل مفاتيح RSA أو EC أو OKP وقد يأتي ضمن مجموعة JWK ‏(JWKS).

PEM هو مفتاح ASN.1/DER مُرمّز بـ Base64 مع أسطر ترويسة مثل BEGIN PUBLIC KEY أو BEGIN PRIVATE KEY، وهو شائع في TLS وOpenSSL والعديد من حزم SDK.

تقوم هذه الأداة بتحويل المفاتيح في الاتجاهين مع الحفاظ على مادة المفتاح عند اختيار إخراج عام (SPKI) أو خاص (PKCS8). تدعم RSA وEC ‏(P-256/384/521) وOKP ‏(Ed25519/X25519/Ed448/X448)، وكل شيء يعمل محلياً داخل المتصفح.

اختر JWK → PEM عندما تتوقع مكتبة أو بوابة أو أداة CLI ملفات مفاتيح بأسلوب OpenSSL. واختر PEM → JWK عندما تحتاج إلى تضمين مفتاح داخل JWKS أو تمريره عبر إعدادات مبنية على JSON أو استخدامه في بيئات المتصفح أو serverless. يحتفظ تحويل المفتاح الخاص بالمادة الخاصة نفسها، لذلك شارك المخرجات العامة فقط إذا كان هذا كل ما يحتاجه الطرف الآخر.

- استخدم مفتاح JWK/JWKS مع أنظمة لا تقبل إلا PEM.
- صدّر مفاتيح PEM لمكتبات JWT أو بوابات API أو توزيع المفاتيح.
- شارك المفاتيح العامة بأمان دون كشف بيانات المفتاح الخاص.
