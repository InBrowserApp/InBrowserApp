## ما هي عملية التحويل بين JWK و PEM؟

JWK ‏(JSON Web Key) هو تمثيل JSON لمواد المفاتيح يُستخدم في JOSE/JWT ونقاط نهاية JWKS وإعدادات المتصفح أو البيئات serverless. يسهل على البرمجيات قراءته، لكنه أقل قبولًا في أدوات CLI والبنية التحتية التي تتوقع ملفات مفاتيح.

PEM يغلّف بيانات مفاتيح DER بعناوين BEGIN/END، وهو الشكل الذي تطلبه عادةً OpenSSL وأدوات TLS وبوابات API وكثير من SDKs.

يربط هذا المحوّل بين التنسيقين محليًا داخل متصفحك. يدعم حاويات مفاتيح RSA وEC ‏(P-256/384/521) وOKP، ويتيح اختيار PEM عام بصيغة SPKI أو خاص بصيغة PKCS8 عند البدء من JWK، كما يحوّل كتل PEM المدعومة إلى JSON JWK منسق أو مضغوط.

استخدم الإخراج العام عندما تحتاج فقط إلى التحقق أو التوزيع. التحويلات الخاصة تعرض مواد المفتاح الخاص على الشاشة وفي التنزيلات، لذلك تعامل مع النتيجة كسر وأغلق التبويب عند الانتهاء.

- انقل المفاتيح بين JWKS/إعدادات JSON وملفات PEM بأسلوب OpenSSL.
- استخرج مفتاحًا عامًا قبل مشاركته مع محققات JWT أو بوابات API أو العملاء.
- حوّل محليًا دون رفع مواد المفاتيح إلى خادم.
