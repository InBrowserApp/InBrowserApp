## מהי המרה בין JWK ל‑PEM?

JWK ‏(JSON Web Key) הוא פורמט JSON למפתחות קריפטוגרפיים המשמש במערכות JOSE/JWT. הוא יכול לייצג מפתחות RSA, EC או OKP ויכול להופיע בתוך JWK Set ‏(JWKS).

PEM הוא מפתח ASN.1/DER שמקודד ב‑Base64 עם שורות כותרת כמו BEGIN PUBLIC KEY או BEGIN PRIVATE KEY, והוא נפוץ ב‑TLS, OpenSSL וב‑SDK רבים.

הכלי ממיר מפתחות לשני הכיוונים, תוך שמירה על חומר המפתח בעת בחירת פלט ציבורי (SPKI) או פרטי (PKCS8). נתמכים RSA, EC ‏(P-256/384/521) ו‑OKP ‏(Ed25519/X25519/Ed448/X448), והכול פועל מקומית בדפדפן.

בחרו ב־JWK → PEM כאשר ספרייה, שער או כלי CLI מצפים לקובצי מפתח בסגנון OpenSSL. בחרו ב־PEM → JWK כאשר צריך לשלב מפתח בתוך JWKS, להעביר אותו דרך תצורה מבוססת JSON או להשתמש בו בסביבת דפדפן או serverless. המרה של מפתח פרטי שומרת על החומר הפרטי, לכן שתפו רק פלט ציבורי אם זה כל מה שהצרכן צריך.

- השתמשו במפתח JWK/JWKS במערכות שמקבלות רק PEM.
- ייצאו מפתחות PEM לספריות JWT, לשערי API או להפצת מפתחות.
- שתפו מפתחות ציבוריים בבטחה בלי לחשוף נתוני מפתח פרטי.
