जब आपको डेटाबेस पंक्तियों, API फिक्स्चर, ऑब्जेक्ट की, टेस्ट पेलोड, इम्पोर्ट टेम्पलेट या एकबारगी संचालन कार्य के लिए यादृच्छिक ID चाहिए हों, तो सीधे अपने ब्राउजर में UUID v4 पहचानकर्ताओं का बैच जनरेट करें.

## What UUID v4 Provides

UUID v4 एक 128-बिट पहचानकर्ता है, जो अधिकतर क्रिप्टोग्राफिक रूप से सुरक्षित यादृच्छिक बाइट्स से बनता है. वर्जन और वैरिएंट बिट RFC 4122 लेआउट से तय होते हैं, इसलिए UUID v4 का परिचित `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx` आकार होता है और फिर भी उसमें बहुत बड़ा रैंडम स्पेस रहता है.

## Pick A Practical Batch Size

डिफॉल्ट बैच आपको कई फिक्स्चर और स्प्रेडशीट वर्कफ्लो के लिए पर्याप्त ID देता है, बिना पेज को स्कैन करना मुश्किल बनाए. जब आप बड़ा इम्पोर्ट तैयार कर रहे हों तो संख्या बढ़ाएं, या जब आपको किसी अनुरोध सैंपल या मैनुअल डेटाबेस एडिट के लिए केवल कुछ पहचानकर्ताओं की जरूरत हो तो इसे घटाएं.

## Copy Or Export

जनरेट की गई सूची की समीक्षा करें, फिर उसे अपने एडिटर में कॉपी करें या एक प्लेन-टेक्स्ट फाइल डाउनलोड करें. हर मान स्थानीय रूप से जनरेट होता है, और मौजूदा बैच इस टूल द्वारा कभी अपलोड नहीं किया जाता.

## Collision Guidance

सामान्य एप्लिकेशन वर्कलोड के लिए UUID v4 collision जोखिम बेहद कम होता है, लेकिन यह डेटाबेस uniqueness constraint का विकल्प नहीं है. जहां ID primary key, public token या durable reference बनती है, वहां uniqueness लागू करते रहें.
