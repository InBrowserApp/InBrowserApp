データベース行、API フィクスチャ、オブジェクトキー、テストペイロード、インポート用テンプレート、一度きりの運用作業にランダム ID が必要なとき、UUID v4 識別子のバッチをブラウザー内で直接生成できます。

## What UUID v4 Provides

UUID v4 は、主に暗号学的に安全なランダムバイトから作られる 128 ビット識別子です。バージョンビットとバリアントビットは RFC 4122 のレイアウトで固定されるため、UUID v4 は見慣れた `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx` 形式を保ちながら、非常に大きなランダム空間を持ちます。

## Pick A Practical Batch Size

既定のバッチは、ページを見づらくせずに、多くのフィクスチャやスプレッドシート作業に十分な数の ID を提供します。大きなインポートを準備するときは件数を増やし、リクエスト例や手動のデータベース編集に少数の識別子だけが必要なときは件数を減らしてください。

## Copy Or Export

生成された一覧を確認してから、エディターにコピーするか、プレーンテキストファイルとしてダウンロードします。すべての値はローカルで生成され、このツールが現在のバッチをアップロードすることはありません。

## Collision Guidance

UUID v4 の衝突リスクは通常のアプリケーション負荷では極めて低いものの、データベースの一意性制約の代わりにはなりません。ID が主キー、公開トークン、永続的な参照になる場所では、一意性の強制を維持してください。
