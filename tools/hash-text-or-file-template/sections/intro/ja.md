## テキストまたはファイルのハッシュとは？

ハッシュ関数は、テキストやファイルのバイト列を固定長のダイジェストに変換します。同じ入力とアルゴリズムは常に同じダイジェストを生成するため、個人データをアップロードせずに再現可能なフィンガープリントが必要な場面でハッシュが役立ちます。

## このツールを使う場面

このツールは、ダウンロードのチェックサム確認、2 つのファイルが同一かどうかの比較、テキスト断片の簡単なフィンガープリント記録、SHA ダイジェストを公開するシステムのデバッグに使用できます。ファイルを読み込むとファイルのバイト列を直接ハッシュ化し、テキストモードではエディターに表示されている UTF-8 テキストをハッシュ化します。

## アルゴリズムの選び方

SHA-256 は、新しい整合性チェックの堅実な既定値です。SHA-384 と SHA-512 は、別のシステムがそれらの形式を要求する場合に、より長い SHA-2 ダイジェストを提供します。SHA-1 はレガシー比較のために含まれていますが、新しいセキュリティ重視の設計には使用しないでください。

## プライバシーと制限事項

ハッシュ化は Web Crypto を通じてブラウザー内でローカルに実行され、ファイルはアップロードされません。ハッシュは暗号化ではありません。それだけで秘密を保護することはできず、パスワード保存にはソルトとワークファクターを備えた専用のパスワードハッシュ関数が必要です。
