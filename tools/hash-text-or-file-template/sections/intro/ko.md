## 텍스트 또는 파일 해시란 무엇인가요?

해시 함수는 텍스트나 파일 바이트를 고정 길이 다이제스트로 변환합니다. 같은 입력과 알고리즘은 항상 같은 다이제스트를 만들기 때문에, 비공개 데이터를 업로드하지 않고 반복 가능한 지문이 필요할 때 유용합니다.

## 이 도구를 사용할 때

이 도구를 사용해 다운로드 체크섬을 확인하고, 두 파일이 동일한지 비교하고, 텍스트 조각의 간단한 지문을 기록하거나, SHA 다이제스트를 게시하는 시스템을 디버그할 수 있습니다. 파일을 가져오면 파일 바이트를 직접 해시하고, 텍스트 모드에서는 편집기에 표시된 UTF-8 텍스트를 해시합니다.

## 알고리즘 선택

SHA-256은 새 무결성 검사의 안정적인 기본값입니다. SHA-384와 SHA-512는 다른 시스템에서 해당 형식을 요구할 때 더 긴 SHA-2 다이제스트를 제공합니다. SHA-1은 레거시 비교를 위해 포함되어 있지만, 새 보안 민감 설계에는 사용해서는 안 됩니다.

## 개인정보 보호 및 제한 사항

해싱은 Web Crypto를 통해 브라우저에서 로컬로 실행되며 파일은 업로드되지 않습니다. 해시는 암호화가 아닙니다. 해시만으로는 비밀을 보호할 수 없으며, 비밀번호 저장에는 솔트와 작업 비용이 있는 전용 비밀번호 해싱 함수가 필요합니다.
