## Unicode Escaping คืออะไร?

Unicode escaping คือการแปลงอักขระให้เป็นลำดับที่เข้ารหัสแล้ว ซึ่งแทนค่า Unicode code point ของอักขระนั้น สิ่งนี้จำเป็นเมื่อซอร์สโค้ด ไฟล์คอนฟิก หรือรูปแบบข้อมูลไม่สามารถบรรจุอักขระบางตัวได้โดยตรง

**รูปแบบการเข้ารหัสที่พบบ่อย:**

- `\uXXXX` — JavaScript / JSON ใช้ในภาษาโปรแกรมส่วนใหญ่
- `\u{XXXXX}` — ES6+ JavaScript รองรับอักขระเสริมโดยไม่ต้องใช้ surrogate pair
- `&#xXXXX;` / `&#DDDD;` — HTML entity ในรูปแบบเลขฐานสิบหกหรือฐานสิบ
- `U+XXXX` — สัญกรณ์ Unicode มาตรฐานที่ใช้ในเอกสาร
- `\xXX` / `%XX` — การเข้ารหัสระดับไบต์ UTF-8 พบได้ทั่วไปใน URL และภาษาตระกูล C
- `\UXXXXXXXX` — รูปแบบ 8 หลักของ Python สำหรับ code point ใดก็ได้
- `0xXXXX` — สัญกรณ์ค่าเลขฐานสิบหก

## เมื่อไหร่ควรใช้เครื่องมือนี้

- ฝังอักขระที่ไม่ใช่ ASCII ในซอร์สโค้ดหรือไฟล์คอนฟิกที่ต้องการการเข้ารหัสแบบ ASCII-safe
- แก้ไขข้อบกพร่องของข้อความที่แสดงผิดเพี้ยน โดยตรวจสอบ Unicode code point ที่อยู่เบื้องหลัง
- แปลงระหว่างรูปแบบการเข้ารหัสต่าง ๆ เมื่อย้ายโค้ดระหว่างภาษาหรือรูปแบบข้อมูล
- เตรียมข้อความสำหรับบริบท JSON, HTML หรือ URL ที่ต้องการอักขระแบบ entity-encoded

## วิธีการทำงาน

พิมพ์หรือวางข้อความธรรมดาที่ด้านซ้าย เครื่องมือจะเข้ารหัสอักขระที่ไม่ใช่ ASCII โดยใช้รูปแบบที่เลือก วางข้อความที่เข้ารหัสแล้วที่ด้านขวา เครื่องมือจะตรวจจับและถอดรหัสรูปแบบที่รองรับทั้งหมดโดยอัตโนมัติ ทุกอย่างทำงานภายในเบราว์เซอร์ของคุณ
