## วิธีแปลง IPv6 เป็นที่อยู่ MAC

คุณจะย้อนกลับจากที่อยู่ IPv6 ไปหา MAC address ได้ก็ต่อเมื่อ interface
identifier ของ IPv6 ถูกสร้างจาก MAC address นั้นด้วยวิธี EUI-64 เท่านั้น
กรณีนี้พบได้บ่อยกับที่อยู่ link-local รุ่นเก่าที่ขึ้นต้นด้วย `fe80::`
และที่อยู่แบบ stateless autoconfiguration บางส่วน

### ใช้ได้เมื่อใด

การแปลงย้อนกลับนี้จะทำงานเมื่อ 64 บิตท้ายของที่อยู่ IPv6 ยังคงมี
interface identifier แบบ EUI-64 อยู่

- Interface identifier ถูกสร้างจาก MAC address ขนาด 48 บิต
- ไบต์ตรงกลางยังคงเป็น `ff:fe`
- ที่อยู่ไม่ได้ถูกสร้างด้วย privacy extensions หรือกลไกสุ่มแบบอื่น

### หลักการแปลง

ตัวแปลงจะสร้าง MAC address กลับมาด้วยขั้นตอนต่อไปนี้:

1. อ่าน 64 บิตท้ายของที่อยู่ IPv6
2. เอาไบต์ `ff:fe` ที่ถูกแทรกไว้กลาง interface identifier ออก
3. กลับค่า universal/local bit ของไบต์แรก
4. จัดรูปแบบ 48 บิตที่เหลือให้เป็น MAC address มาตรฐาน

### ทำไมจึงไม่มีผลลัพธ์

คุณอาจไม่ได้ผลลัพธ์ด้วยสาเหตุต่อไปนี้:

- รูปแบบของที่อยู่ IPv6 ไม่ถูกต้อง
- ที่อยู่ถูกต้อง แต่ไม่ได้ถูกสร้างจาก MAC address ด้วย EUI-64
- ที่อยู่ใช้ privacy, stable-random, DHCPv6 หรือวิธีแจกจ่ายที่ไม่อิง MAC
