## Что такое JSON Schema?

JSON Schema — это стандарт для описания структуры данных JSON. Он позволяет задавать типы полей, вложенные структуры, обязательные ключи и ограничения, удобные для валидации.

### Что делает этот генератор

Вставьте пример JSON, и инструмент выведет начальную schema для объектов, массивов, чисел, булевых значений, null и распространённых строковых форматов. Результат можно копировать, скачивать и дорабатывать.

### Пример

Например, для такого примера данных:

**Пример входных данных**

```json
{
  "id": "bk-101",
  "title": "In-browser Tools",
  "price": 19.99,
  "tags": ["json", "schema"],
  "published": true
}
```

**Сгенерированная schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "id": { "type": "string" },
    "title": { "type": "string" },
    "price": { "type": "number" },
    "tags": {
      "type": "array",
      "items": { "type": "string" }
    },
    "published": { "type": "boolean" }
  },
  "required": ["id", "title", "price", "tags", "published"]
}
```

### Советы

- Используйте репрезентативные примеры данных, особенно внутри массивов, чтобы проще выявлять необязательные поля.
- Отключите «Infer required properties», если входные данные — только неполный пример.
- Отключите «Allow additional properties», если нужен более строгий schema по умолчанию.
- Оставьте определение строковых форматов включённым, чтобы распознавать email, URI, UUID и date-time.
