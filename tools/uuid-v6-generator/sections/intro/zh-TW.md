UUID v6 產生器會建立以時間為基礎的 UUID，保留熟悉的 UUID 格式，同時將時間戳記放在前面，方便自然的字典順序排序。它完全在你的瀏覽器中執行，因此產生的識別碼和選用的節點值都不會離開頁面。

## UUID v6 適用時機

當你需要識別碼能與 UUID 工具廣泛相容，又能在記錄、資料庫索引、事件串流或移轉腳本中接近建立順序排序時，請使用 UUID v6。從語意上來看，UUID v6 最接近 UUID v1：它使用格里曆時間戳記、時鐘序列和 48 位元節點欄位，但會重新排列時間戳記位元，讓較新的 ID 排在較舊的 ID 之後。

## 節點 ID 與隱私

傳統 UUID v1 產生器通常會使用真實 MAC 位址作為節點欄位。此工具預設會為每個產生的 UUID 使用隨機且本機管理的節點 ID，因此不會暴露硬體位址。只有在你刻意需要 v1 相容輸出，用於測試資料、互通檢查或受控系統時，才切換到自訂節點。

## 時鐘序列與自訂時間

當時間戳記重複或時鐘往回移動時，時鐘序列有助於避免碰撞。預設的隨機序列對一般用途最安全。自訂時間戳記、節點 ID 和時鐘序列適合用於可重現的範例，但在正式資料中應謹慎使用重複的自訂值。
