# ตัวสร้าง CSR

Certificate Signing Request (CSR) คือข้อความ PKCS#10 ที่ประกอบด้วยคีย์สาธารณะของคุณ ฟิลด์ Subject ที่ใช้ระบุตัวตน ส่วนขยายเสริม เช่น Subject Alternative Names และลายเซ็นที่สร้างด้วยคีย์ส่วนตัวที่ตรงกัน ผู้ออกใบรับรองใช้ CSR เพื่อออกใบรับรอง X.509 โดยไม่ต้องได้รับคีย์ส่วนตัวของคุณ

ตัวสร้างนี้สร้าง CSR โดยตรงในเบราว์เซอร์ของคุณ คุณสามารถสร้างคู่คีย์ RSA หรือ ECDSA ใหม่ หรือนำเข้าคีย์ส่วนตัว PEM ที่มีอยู่และไม่ได้เข้ารหัส เมื่อต้องต่ออายุใบรับรองสำหรับคีย์ที่ใช้งานอยู่แล้ว

## ควรใช้เมื่อใด

ใช้ CSR เมื่อคุณต้องการให้ผู้ออกใบรับรองออกหรือต่ออายุใบรับรอง TLS, S/MIME, การยืนยันตัวตนไคลเอนต์ หรือบริการภายใน CSR พิสูจน์ว่าคุณครอบครองคีย์ส่วนตัวและบรรจุข้อมูลตัวตนสาธารณะที่ควรปรากฏในใบรับรอง

สำหรับใบรับรอง TLS สาธารณะ ให้ใส่ชื่อโฮสต์ไว้ใน Subject Alternative Names Common Name ยังมีประโยชน์ต่อการอ่านและระบบเดิม แต่ไคลเอนต์สมัยใหม่ตรวจสอบชื่อ DNS และที่อยู่ IP จาก SAN

## วิธีสร้าง CSR

เลือกว่าจะสร้างคีย์ใหม่หรือนำเข้าคีย์ส่วนตัวที่มีอยู่ กรอกฟิลด์ Subject ที่จำเป็นสำหรับคำขอใบรับรองของคุณ จากนั้นเพิ่มรายการ SAN สำหรับชื่อ DNS, ที่อยู่ IP, ที่อยู่อีเมล หรือ URI สร้าง CSR แล้วส่งเฉพาะ CSR PEM ให้ผู้ออกใบรับรองของคุณ

หากเครื่องมือนี้สร้างคีย์ใหม่ ให้ดาวน์โหลดและจัดเก็บคีย์ส่วนตัวก่อนออกจากหน้า หากคุณนำเข้าคีย์ เครื่องมือจะสร้างเฉพาะ CSR และจะไม่ส่งออกคีย์ส่วนตัวที่นำเข้าอีกครั้ง

## หมายเหตุเกี่ยวกับคีย์และรูปแบบ

RSA 2048 บิตเข้ากันได้อย่างกว้างขวาง ส่วน 3072 หรือ 4096 บิตอาจเหมาะกว่าสำหรับใบรับรองภายในที่มีอายุใช้งานนาน ECDSA P-256 มีขนาดกะทัดรัดและรองรับอย่างแพร่หลาย ขณะที่ P-384 หรือ P-521 อาจจำเป็นสำหรับนโยบายที่เข้มงวดกว่า เส้นทางการนำเข้าคีย์รองรับบล็อก PEM แบบ PKCS#8, RSA PRIVATE KEY และ EC PRIVATE KEY ที่ไม่ได้เข้ารหัส

คีย์ส่วนตัวเป็นข้อมูลละเอียดอ่อน อย่าวางลงในเว็บไซต์ที่ไม่น่าเชื่อถือ อย่าส่งให้ผู้ออกใบรับรอง และอย่าคอมมิตลงในระบบควบคุมซอร์สโค้ด เครื่องมือนี้ทำงานในเครื่องภายในเบราว์เซอร์ แต่กระบวนการปฏิบัติงานของคุณยังต้องมีการจัดเก็บและหมุนเวียนคีย์อย่างปลอดภัย
