# CSR 생성기

인증서 서명 요청(CSR)은 공개 키, 식별용 Subject 필드, Subject Alternative Names 같은 선택적 확장, 그리고 일치하는 개인 키로 만든 서명을 포함하는 PKCS#10 메시지입니다. 인증 기관은 CSR을 사용해 개인 키를 전달받지 않고도 X.509 인증서를 발급합니다.

이 생성기는 브라우저에서 직접 CSR을 만듭니다. 새 RSA 또는 ECDSA 키 쌍을 생성할 수 있고, 이미 배포된 키로 인증서를 갱신해야 할 때는 기존의 암호화되지 않은 PEM 개인 키를 가져올 수 있습니다.

## 사용 시점

TLS, S/MIME, 클라이언트 인증 또는 내부 서비스 인증서를 인증 기관에서 발급하거나 갱신해야 할 때 CSR을 사용합니다. CSR은 개인 키를 보유하고 있음을 증명하며, 인증서에 표시되어야 할 공개 신원 정보를 담습니다.

공개 TLS 인증서의 경우 호스트 이름은 Subject Alternative Names에 넣으세요. Common Name은 가독성과 레거시 시스템을 위해 여전히 유용하지만, 최신 클라이언트는 SAN의 DNS 이름과 IP 주소를 검증합니다.

## CSR 생성 방법

새 키를 생성할지 기존 개인 키를 가져올지 선택합니다. 인증서 요청에 필요한 Subject 필드를 입력한 다음 DNS 이름, IP 주소, 이메일 주소 또는 URI에 대한 SAN 항목을 추가합니다. CSR을 생성하고 CSR PEM만 인증 기관에 보내세요.

이 도구가 새 키를 생성한 경우 페이지를 떠나기 전에 개인 키를 다운로드해 보관하세요. 키를 가져온 경우 도구는 CSR만 생성하며 가져온 개인 키를 다시 내보내지 않습니다.

## 키 및 형식 참고 사항

RSA 2048비트는 폭넓게 호환되며, 수명이 긴 내부 인증서에는 3072비트 또는 4096비트가 더 적합할 수 있습니다. ECDSA P-256은 작고 널리 지원되며, 더 엄격한 정책에서는 P-384 또는 P-521이 필요할 수 있습니다. 가져오기 경로는 암호화되지 않은 PKCS#8, RSA PRIVATE KEY, EC PRIVATE KEY PEM 블록을 지원합니다.

개인 키는 민감한 정보입니다. 신뢰할 수 없는 웹사이트에 붙여넣지 말고, 인증 기관에 보내지 말고, 소스 제어에 커밋하지 마세요. 이 도구는 브라우저에서 로컬로 실행되지만, 운영 프로세스에서는 여전히 안전한 키 저장과 교체가 필요합니다.
