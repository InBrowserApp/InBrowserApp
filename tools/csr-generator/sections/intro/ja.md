# CSR Generator

証明書署名要求 (CSR) は PKCS#10 メッセージで、公開鍵、識別用の Subject フィールド、Subject Alternative Names などの任意の拡張、対応する秘密鍵で作成された署名を含みます。認証局は CSR を使って、秘密鍵を受け取ることなく X.509 証明書を発行します。

このジェネレーターはブラウザー内で直接 CSR を作成します。新しい RSA または ECDSA 鍵ペアを生成できます。また、すでに展開済みの鍵で証明書を更新する必要がある場合は、既存の暗号化されていない PEM 秘密鍵をインポートできます。

## When to use it

TLS、S/MIME、クライアント認証、または内部サービス用の証明書を認証局に発行または更新してもらう必要があるときに CSR を使います。CSR は秘密鍵を保持していることを証明し、証明書に記載すべき公開の識別情報を運びます。

公開 TLS 証明書では、ホスト名を Subject Alternative Names に入れてください。Common Name は可読性やレガシーシステムのために今でも役立ちますが、最新のクライアントは SAN の DNS 名と IP アドレスを検証します。

## How to generate a CSR

新しい鍵を生成するか、既存の秘密鍵をインポートするかを選択します。証明書要求に必要な Subject フィールドを入力し、DNS 名、IP アドレス、メールアドレス、URI の SAN エントリーを追加します。CSR を生成し、CSR PEM のみを認証局に送信します。

このツールで新しい鍵を生成した場合は、ページを離れる前に秘密鍵をダウンロードして保管してください。鍵をインポートした場合、このツールは CSR のみを生成し、インポートした秘密鍵は再エクスポートしません。

## Key and format notes

RSA 2048 ビットは広く互換性があります。より長期間使用する内部証明書では 3072 または 4096 ビットが好まれる場合があります。ECDSA P-256 はコンパクトで広くサポートされていますが、より厳格なポリシーでは P-384 または P-521 が必要になることがあります。鍵インポートでは、暗号化されていない PKCS#8、RSA PRIVATE KEY、EC PRIVATE KEY の PEM ブロックに対応しています。

秘密鍵は機密情報です。信頼できない Web サイトに貼り付けたり、認証局に送信したり、ソース管理にコミットしたりしないでください。このツールはブラウザー内でローカルに動作しますが、運用プロセスでは引き続き安全な鍵保管とローテーションが必要です。
