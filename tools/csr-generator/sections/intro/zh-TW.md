# CSR 產生器

憑證簽署要求 (CSR) 是一則 PKCS#10 訊息，包含你的公開金鑰、用於識別的 Subject 欄位、Subject Alternative Names 等選用擴充，以及使用相符私密金鑰產生的簽章。憑證機構會使用 CSR 簽發 X.509 憑證，而不需要取得你的私密金鑰。

此產生器會直接在你的瀏覽器中建立 CSR。你可以產生新的 RSA 或 ECDSA 金鑰組，或在需要為已部署金鑰續期憑證時，匯入現有未加密的 PEM 私密金鑰。

## 何時使用

當你需要憑證機構簽發或續期 TLS、S/MIME、用戶端驗證或內部服務憑證時，請使用 CSR。CSR 可證明你持有私密金鑰，並帶有應出現在憑證中的公開身分資訊。

對於公開 TLS 憑證，請將主機名稱放在 Subject Alternative Names 中。Common Name 仍有助於可讀性與舊系統相容，但現代用戶端會從 SAN 驗證 DNS 名稱與 IP 位址。

## 如何產生 CSR

選擇要產生全新金鑰，或匯入現有私密金鑰。填寫憑證要求所需的 Subject 欄位，然後為 DNS 名稱、IP 位址、電子郵件地址或 URI 新增 SAN 項目。產生 CSR 後，只將 CSR PEM 傳送給你的憑證機構。

如果此工具產生了新金鑰，請在離開頁面前下載並保存私密金鑰。如果你匯入金鑰，工具只會產生 CSR，不會重新匯出匯入的私密金鑰。

## 金鑰與格式注意事項

RSA 2048 位元具備廣泛相容性；3072 或 4096 位元可能更適合生命週期較長的內部憑證。ECDSA P-256 精簡且支援廣泛，而 P-384 或 P-521 可能是更嚴格政策的要求。匯入金鑰流程支援未加密的 PKCS#8、RSA PRIVATE KEY 與 EC PRIVATE KEY PEM 區塊。

私密金鑰屬於敏感資料。不要將它們貼到不受信任的網站、不要傳送給憑證機構，也不要提交到原始碼控制系統。此工具會在瀏覽器中本機執行，但你的作業流程仍需要安全的金鑰儲存與輪替。
