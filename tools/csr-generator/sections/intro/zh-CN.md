# CSR 生成器

证书签名请求（CSR）是一条 PKCS#10 消息，包含你的公钥、用于标识身份的 Subject 字段、Subject Alternative Names 等可选扩展，以及使用匹配私钥生成的签名。证书颁发机构使用 CSR 签发 X.509 证书，而无需接收你的私钥。

此生成器会直接在浏览器中创建 CSR。你可以生成新的 RSA 或 ECDSA 密钥对，也可以在需要为已部署密钥续期证书时，导入现有的未加密 PEM 私钥。

## 何时使用

当你需要证书颁发机构签发或续期 TLS、S/MIME、客户端身份验证或内部服务证书时，请使用 CSR。CSR 证明你拥有私钥，并携带应出现在证书中的公开身份信息。

对于公共 TLS 证书，请将主机名放入 Subject Alternative Names。Common Name 对可读性和旧系统仍然有用，但现代客户端会根据 SAN 中的 DNS 名称和 IP 地址进行验证。

## 如何生成 CSR

选择生成新密钥或导入现有私钥。填写与你的证书请求相关的 Subject 字段，然后为 DNS 名称、IP 地址、电子邮件地址或 URI 添加 SAN 条目。生成 CSR 后，只将 CSR PEM 发送给你的证书颁发机构。

如果此工具生成了新密钥，请在离开页面前下载并保存私钥。如果你导入密钥，工具只会生成 CSR，不会重新导出导入的私钥。

## 密钥和格式说明

RSA 2048 位兼容性广；对于有效期更长的内部证书，可能更适合使用 3072 或 4096 位。ECDSA P-256 体积小且支持广泛，而更严格的策略可能要求 P-384 或 P-521。导入密钥路径支持未加密的 PKCS#8、RSA PRIVATE KEY 和 EC PRIVATE KEY PEM 块。

私钥属于敏感信息。不要将其粘贴到不受信任的网站，不要发送给证书颁发机构，也不要提交到源代码管理。此工具在浏览器本地运行，但你的运维流程仍需要安全的密钥存储和轮换。
