## Generated paths, localized payloads

The tools index no longer needs a hand-written route table. Static paths come from `packages/tool-registry`, while localized metadata and long-form content stay with each tool directory.

That keeps the contract small:

- `manifest.ts` defines the tool identity
- `messages/<lang>.json` owns localized labels and metadata
- `content/<lang>.mdx` owns long-form explanations
- optional client islands stay in tool-local modules

Once the first migrated tools land, this page will render their localized detail routes without any extra registry wiring.
