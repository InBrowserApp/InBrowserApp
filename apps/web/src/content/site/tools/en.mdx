## Generated paths now point to self-contained pages

The tools index no longer needs a hand-written route table, and it no longer needs to understand each tool's internal content structure. Static paths come from `packages/tool-registry`, while localized discovery data comes from each tool's `meta/` directory.

That keeps the platform contract small:

- `manifest.ts` defines non-localized discovery metadata
- `index.astro` is the fixed tool entrypoint
- `meta/<lang>.json` provides localized names and descriptions

Inside that boundary, each tool is free to organize its own copy, markdown sections, client code, and private helpers however it wants.
