## 所有页面共用一套路由模型

新的 Astro 重写把本地化首先当成一个路由问题来处理。英文页面保留在根路径下，非默认语言通过 `/:lang` 前缀进入，但继续复用同一套页面组件。

这样 canonical URL、alternate language links 和 sitemap 就能直接从真实页面树推导出来，而不是再依赖运行时 router alias。

## 内容跟随页面职责放置

短文案继续放在 JSON message catalog 里，长说明内容放在各语言对应的 MDX 文件里。这样 UI 文案、SEO 元数据和编辑型内容不会重新堆回一个组件文件。

同一套 fallback 规则现在同时适用于站点页面和工具页面，后续工具接入多语言内容时也不需要再各自发明加载方式。
