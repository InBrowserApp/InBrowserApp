## The shell now hosts real tools

The rewrite is no longer just proving the shell. The first migrated tools now render through the generated registry, which means the same localization model is exercising real text, validation, and file-processing flows.

That matters because a route tree looks stable long before the interaction model is. Shipping the first archetypes proves that manifests, localized assets, page entrypoints, and islands can actually move together.

## Tool-local pages are now part of the contract

Each migrated tool now owns:

- `manifest.ts` for identity
- `index.astro` for page-level composition
- `client.tsx` for interaction
- `messages/<lang>.json` for short labels
- `content/<lang>.mdx` for long-form context

That keeps tool code self-contained while still allowing the site shell to generate routes, search data, and metadata automatically.
