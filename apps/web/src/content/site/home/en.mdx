## The shell now hosts real, self-contained tools

The rewrite is no longer just proving the shell. The first migrated tools now render through the generated registry, which means the same platform layer is exercising real text, validation, and file-processing flows.

That matters because a route tree can look stable long before the tool boundary is actually maintainable. Shipping the first archetypes proves that a smaller contract can still power discovery, search, and localized detail pages.

## The contract is smaller now

Each migrated tool now guarantees only a few things to the shell:

- `manifest.ts` for non-localized discovery metadata
- `index.astro` as the fixed page entrypoint
- `meta/<lang>.json` for localized name and description

Everything else stays inside the tool. A tool may use `client.tsx`, `messages/`, `sections/`, `components/`, or `workers/`, but those are no longer shell-level contract fields.
